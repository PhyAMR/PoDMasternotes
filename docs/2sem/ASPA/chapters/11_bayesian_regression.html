12.1 Introduction

Regression analysis seeks to learn a functional relationship between input variables \(x\) and observed response variables \(y\) from data. In the Bayesian framework, we construct a probabilistic model that combines a generative (noise-free) function \(f(x; \theta)\) with a noise model, then use Bayes’ theorem to infer the posterior distribution \(p(\theta \mid D)\) of the parameters. For regression models where the parameters enter linearly, exact posterior inference is tractable via conjugate analysis. For nonlinear models or non-Gaussian likelihoods, we rely on Markov chain Monte Carlo (MCMC) sampling. This chapter develops both the theory and practice of Bayesian regression: from the closed-form solution for linear Gaussian regression to the complete MCMC workflow for fitting arbitrary nonlinear regression models and making probabilistic predictions.

12.2 Regression Model: Signal Plus Noise

12.2.1 Model Formulation

The basic regression model specifies that each observed response \(y_i\) arises from a noise-free mean function plus random noise:

\[y_i = f(x_i; \theta) + \varepsilon_i, \quad \varepsilon_i \sim \mathcal{N}(0, \sigma^2).\]

Here: - \(x_i\) is the regressor (input, explanatory variable), assumed fixed and measured without error. - \(y_i\) is the response (dependent, outcome variable). - \(f(x_i; \theta)\) is the signal—the deterministic part. - \(\varepsilon_i\) is measurement noise or intrinsic scatter, i.i.d. Gaussian with variance \(\sigma^2\).

12.2.2 Likelihood for Gaussian Noise

Under independent Gaussian noise, each observation has conditional distribution \[p(y_i \mid x_i, \theta) = \mathcal{N}(f(x_i; \theta), \sigma^2).\]

For \(N\) independent observations \(D = \{(x_i, y_i)\}_{i=1}^N\), the joint likelihood factorizes: \[p(D \mid \theta) = \prod_{i=1}^N p(y_i \mid x_i, \theta) = \prod_{i=1}^N \mathcal{N}(f(x_i; \theta), \sigma^2).\]

Taking logarithms, \[\log p(D \mid \theta) = -N \log\sigma - \frac{1}{2}\log(2\pi) - \frac{1}{2\sigma^2} \sum_{i=1}^N [y_i - f(x_i; \theta)]^2 + \text{const}.\]

12.3 Linear Models and the Design Matrix

12.3.1 Linearity in Parameters

A regression model is linear if the parameters enter linearly, even if the features depend nonlinearly on the inputs:

\[y_i = \boldsymbol{\phi}(x_i)^\top \mathbf{w} + \varepsilon_i,\]

where \(\boldsymbol{\phi}(x) = [\phi_1(x), \ldots, \phi_M(x)]^\top\) is a feature map and \(\mathbf{w} = [w_0, \ldots, w_{M-1}]^\top\) is the weight vector.

Examples of linear regression models: - Straight line: \(\boldsymbol{\phi}(x) = (1, x)\), \(f(x) = w_0 + w_1 x\). - Polynomial: \(\boldsymbol{\phi}(x) = (1, x, x^2, \ldots, x^{M-1})\), \(f(x) = \sum_{j=0}^{M-1} w_j x^j\). - Fourier: \(\boldsymbol{\phi}(x) = (1, \sin(x), \cos(x), \ldots)\). - Linear in unknown parameters: If \(f(x) = w_0 e^{-\lambda x}\) with \(\lambda\) known, it is linear in \(w_0\). - Not linear in unknown parameters: \(f(x) = w_0 e^{-\lambda x}\) if \(\lambda\) is unknown (exponential, logistic, etc.).

12.3.2 Design Matrix Notation

For \(N\) data points and \(M\) basis functions, define the \(N \times M\) design matrix (or regressor matrix) \[\boldsymbol{\Phi} = \begin{bmatrix} \phi_1(x_1) & \phi_2(x_1) & \cdots & \phi_M(x_1) \\ \phi_1(x_2) & \phi_2(x_2) & \cdots & \phi_M(x_2) \\ \vdots & \vdots & \ddots & \vdots \\ \phi_1(x_N) & \phi_2(x_N) & \cdots & \phi_M(x_N) \end{bmatrix}.\]

The linear model becomes \[\mathbf{y} = \boldsymbol{\Phi} \mathbf{w} + \boldsymbol{\varepsilon}, \quad \boldsymbol{\varepsilon} \sim \mathcal{N}(\mathbf{0}, \sigma^2 \mathbf{I}_N),\]

and the likelihood is \[p(\mathbf{y} \mid \boldsymbol{\Phi}, \mathbf{w}, \sigma^2) = \mathcal{N}(\mathbf{y} \mid \boldsymbol{\Phi} \mathbf{w}, \sigma^2 \mathbf{I}_N).\]

12.4 Bayesian Linear Regression: Closed-Form Solution

12.4.1 Conjugate Prior and Posterior

Assume \(\sigma^2\) is known. Choose a Gaussian prior on \(\mathbf{w}\): \[p(\mathbf{w}) = \mathcal{N}(\mathbf{w} \mid \mathbf{m}_0, \mathbf{S}_0).\]

The likelihood is Gaussian in \(\mathbf{w}\); by conjugacy, the posterior is also Gaussian:

Bayesian Linear Regression with Gaussian Prior:

Given likelihood \(p(\mathbf{y} \mid \boldsymbol{\Phi}, \mathbf{w}, \sigma^2) = \mathcal{N}(\mathbf{y} \mid \boldsymbol{\Phi} \mathbf{w}, \sigma^2 \mathbf{I}_N)\) and prior \(\mathcal{N}(\mathbf{w} \mid \mathbf{m}_0, \mathbf{S}_0)\), the posterior is \[p(\mathbf{w} \mid D, \sigma^2) = \mathcal{N}(\mathbf{w} \mid \mathbf{m}_N, \mathbf{S}_N),\]

where \[\begin{align} \mathbf{S}_N^{-1} &= \mathbf{S}_0^{-1} + \sigma^{-2} \boldsymbol{\Phi}^\top \boldsymbol{\Phi}, \\ \mathbf{m}_N &= \mathbf{S}_N \left( \mathbf{S}_0^{-1} \mathbf{m}_0 + \sigma^{-2} \boldsymbol{\Phi}^\top \mathbf{y} \right). \end{align}\]

Interpretation: Precisions add. The posterior mean is a precision-weighted average of the prior mean and the data-driven estimate. The posterior covariance \(\mathbf{S}_N\) shrinks relative to \(\mathbf{S}_0\).

12.4.2 Special Case: Weak Isotropic Prior

A common choice is the weak prior \(\mathbf{m}_0 = \mathbf{0}\) and \(\mathbf{S}_0 = \sigma_0^2 \mathbf{I}_M\): \[\mathbf{S}_N^{-1} = \frac{1}{\sigma_0^2} \mathbf{I}_M + \sigma^{-2} \boldsymbol{\Phi}^\top \boldsymbol{\Phi},\] \[\mathbf{m}_N = \sigma^{-2} \mathbf{S}_N \boldsymbol{\Phi}^\top \mathbf{y}.\]

12.4.3 Connection to Ridge Regression

Ordinary least squares minimizes \(\|\mathbf{y} - \boldsymbol{\Phi} \mathbf{w}\|^2\). Ridge regression adds an L2 penalty: \[L_{\text{ridge}} = \|\mathbf{y} - \boldsymbol{\Phi} \mathbf{w}\|^2 + \lambda \|\mathbf{w}\|^2.\]

The ridge estimator is \(\hat{\mathbf{w}}_{\text{ridge}} = (\boldsymbol{\Phi}^\top \boldsymbol{\Phi} + \lambda \mathbf{I})^{-1} \boldsymbol{\Phi}^\top \mathbf{y}\).

Ridge regression is maximum a posteriori (MAP) estimation under a Gaussian prior: the prior variance relates to the penalty by \(\lambda = \sigma^2 / \sigma_0^2\). The Bayesian approach goes further by quantifying posterior uncertainty via the full covariance \(\mathbf{S}_N\).

12.5 Posterior Predictive Distribution

12.5.1 Parameter Uncertainty and Prediction Uncertainty

For a new input $x_* $, we wish to predict \(y_*\). The posterior predictive distribution marginalizes over parameter uncertainty: \[p(y_* \mid x_*, D, \sigma^2) = \int p(y_* \mid x_*, \mathbf{w}, \sigma^2) \, p(\mathbf{w} \mid D, \sigma^2) \, d\mathbf{w}.\]

Since both terms are Gaussian, the integral is tractable:

Posterior Predictive Distribution:

Let \(\boldsymbol{\phi}_* = \boldsymbol{\phi}(x_*)\). Then \[p(y_* \mid x_*, D, \sigma^2) = \mathcal{N}\left( y_* \,\big|\, \boldsymbol{\phi}_*^\top \mathbf{m}_N, \, \sigma^2 + \boldsymbol{\phi}_*^\top \mathbf{S}_N \boldsymbol{\phi}_* \right).\]

Two sources of uncertainty in prediction: 1. Observation noise: \(\sigma^2\) (irreducible error in \(y_*\)). 2. Parameter uncertainty: \(\boldsymbol{\phi}_*^\top \mathbf{S}_N \boldsymbol{\phi}_*\) (uncertainty in the regression function \(f(x_*)\)).

The posterior predictive band widens away from the data due to increasing parameter uncertainty.

12.6 Unknown Noise Level

In practice, \(\sigma^2\) is usually unknown and inferred from data. The parameters become \(\boldsymbol{\theta} = (\mathbf{w}, \sigma)\), and we use a prior on the noise scale. A standard choice is the Jeffreys prior for scale parameters: \[p(\sigma) \propto \frac{1}{\sigma}, \quad \text{or equivalently} \quad p(\log \sigma) \text{ is uniform}.\]

The joint posterior is \[p(\mathbf{w}, \sigma \mid D) \propto p(\mathbf{y} \mid \boldsymbol{\Phi}, \mathbf{w}, \sigma) \, p(\mathbf{w}) \, p(\sigma).\]

When both \(\mathbf{w}\) and \(\sigma\) are unknown, the posterior no longer has a closed form in closed-form distributions, but conjugate analysis can still apply to marginal posteriors or conditional distributions. For computational convenience and to make use of existing MCMC software, we often resort to sampling.

12.7 Nonlinear Regression and MCMC

12.7.1 Why Nonlinear Models Require MCMC

A model is nonlinear in parameters if \(f(x; \theta)\) is a nonlinear function of some \(\theta\). Examples: - Exponential: \(f(x) = a e^{b x}\) (nonlinear in both \(a\) and \(b\) unless one is known). - Logistic: \(f(x) = \frac{L}{1 + e^{-k(x - x_0)}}\) (nonlinear in location \(x_0\), scale \(k\), amplitude \(L\)). - Power law: \(f(x) = a x^b\) (nonlinear in exponent \(b\)).

For such models, the likelihood is no longer Gaussian in the parameters, and conjugate priors do not yield closed-form posteriors. Similarly, generalized linear models (logistic regression for binary data, Poisson regression for counts) have non-Gaussian likelihoods, precluding analytical solutions.

In these cases, we must either: - Use approximations (Laplace, variational inference). - Use MCMC or other sampling methods to draw from \(p(\boldsymbol{\theta} \mid D) \propto p(D \mid \boldsymbol{\theta}) \, p(\boldsymbol{\theta})\).

12.8 Practical MCMC Fitting Workflow

The following seven-step workflow is the standard approach for fitting regression models via MCMC:

12.8.1 Step 1: Specify the Generative Model and Noise Model

Write down: - The regression function: \(f(x; \boldsymbol{\theta})\). - The noise distribution: e.g., \(\varepsilon \sim \mathcal{N}(0, \sigma^2)\). - The observation model: \(y_i \sim \mathcal{N}(f(x_i; \boldsymbol{\theta}), \sigma^2)\).

12.8.2 Step 2: Choose Priors for All Parameters

Specify weakly informative priors for each parameter. For regression coefficients, use broad Gaussians (e.g., \(\mathcal{N}(0, 10^2)\)). For scale parameters like \(\sigma\), use a Jeffreys prior \(p(\sigma) \propto 1/\sigma\) or truncated exponentials. To avoid boundary issues, sample on the log-scale: \(\eta = \log \sigma\).

Reparameterization for Bounded Parameters:

Parameters constrained to positive values (like \(\sigma > 0\)) should be sampled in transformed space: - Replace \(\sigma\) with \(\eta = \log \sigma\), so \(\sigma = e^\eta\). - A flat prior on \(\eta\) corresponds to a Jeffreys prior on \(\sigma\). - The Jacobian \(|d\sigma / d\eta| = e^\eta = \sigma\) automatically cancels the \(1/\sigma\) prior, yielding a uniform prior in log-space.

For slope parameters, reparameterize via angle: \(\beta = \tan(\alpha)\), so all slopes are finite and comparable.

12.8.3 Step 3: Write Log-Prior, Log-Likelihood, and Log-Posterior Functions

Define functions that compute: \[\begin{align} \log p(\boldsymbol{\theta}) &: \text{ sum of log-prior densities for all parameters}, \\ \log p(D \mid \boldsymbol{\theta}) &: \text{ log-likelihood, typically } \sum_{i=1}^N \log p(y_i \mid x_i, \boldsymbol{\theta}), \\ \log p(\boldsymbol{\theta} \mid D) &= \log p(D \mid \boldsymbol{\theta}) + \log p(\boldsymbol{\theta}) + \text{const}. \end{align}\]

For MCMC, we only need the unnormalized posterior (up to an additive constant).

12.8.4 Step 4: Run MCMC and Tune Proposal Covariance

Initialize parameters at a reasonable starting point (e.g., a classical fit or MAP estimate). Specify a proposal distribution, typically a multivariate Gaussian with tuned covariance. Run MCMC (e.g., random-walk Metropolis-Hastings, or preferably NUTS via Stan) for a burn-in period, then for additional sampling iterations.

Tune the proposal variance to achieve a target acceptance rate (typically 20–30% for random-walk MH, automatic for NUTS).

12.8.5 Step 5: Diagnose and Inspect the Chain

Remark. Key Diagnostics: - Trace plots: Plot \(\theta_j\) versus iteration. Traces should resemble white noise (good mixing), not slow drifts or stuck chains. - Autocorrelation function (ACF): Compute \(\text{Corr}(\theta_j^{(k)}, \theta_j^{(k+\ell)})\) as a function of lag \(\ell\). Rapid decay (small effective sample size if slow decay) indicates good mixing. - Effective sample size (ESS): \(\text{ESS} = N_{\text{samples}} / (1 + 2 \sum_{\ell=1}^\infty \text{Corr}(\ell))\). Rule of thumb: aim for ESS \(> 1000\) per parameter. - \(\hat{R}\) (Gelman-Rubin statistic): Ratio of between-chain to within-chain variance. \(\hat{R} < 1.01\) indicates convergence; \(\hat{R} > 1.05\) indicates poor mixing or insufficient burn-in. - Joint posterior pairs plot: Scatter plot of all pairs of parameters. Reveals correlations and identifies poorly identified parameters.

12.8.6 Step 6: Summarize Parameters

Compute posterior summaries from the converged samples (after thinning to reduce autocorrelation if desired): - Maximum a posteriori (MAP): The sample with highest posterior density. - Posterior mean: Average of all samples. - Posterior covariance: Empirical covariance of samples. - Credible intervals: 2.5th and 97.5th quantiles (or other quantiles) per parameter.

Always transform samples back to the original parameterization before computing summaries.

12.8.7 Step 7: Plot Predictions and Posterior Predictive Checks

  1. Data plot with error bars: Show observed \((x_i, y_i)\) with vertical error bars at \(\pm \sigma_{\text{MAP}}\) or \(\pm\) posterior SD of \(y_i\).

  2. Fitted regression curve: Plot the posterior mean fit \(\hat{f}(x) = \boldsymbol{\phi}(x)^\top \mathbb{E}[\mathbf{w} \mid D]\).

  3. Credible band for the mean curve: For a grid of \(x\) values, compute the posterior quantiles of \(f(x; \boldsymbol{\theta})\) by evaluating the function at each posterior sample. This shows uncertainty in the regression function, not including observation noise.

  4. Posterior predictive band: For future observations, include both parameter uncertainty and noise. For each \(x\) on the grid and each posterior sample \(\boldsymbol{\theta}^{(k)}\), generate \(y^{(k)} \sim \mathcal{N}(f(x; \boldsymbol{\theta}^{(k)}), \sigma^{(k)2})\). The 2.5th and 97.5th quantiles across samples form the prediction interval.

12.9 Example: Linear Regression with MCMC

Consider fitting a straight line \(f(x) = b_0 + b_1 x\) to observed data \(\{(x_i, y_i)\}\), with unknown noise \(\sigma\).

Parameters (transformed): \(\boldsymbol{\theta} = (b_0, \alpha, \eta)\) where \(b_1 = \tan(\alpha)\) and \(\eta = \log \sigma\).

Priors: - \(b_0 \sim \mathcal{N}(0, 10^2)\). - \(\alpha \sim \text{Uniform}(-\pi/2, \pi/2)\) (equivalently, \(p(\alpha) \propto 1\)). - \(\eta \sim \text{Uniform}\) (flat prior on \(\log \sigma\), Jeffreys prior on \(\sigma\)).

Log-likelihood: \[\log p(\mathbf{y} \mid b_0, \alpha, \eta) = -N \eta - \frac{1}{2} \sum_{i=1}^N \frac{[y_i - (b_0 + \tan(\alpha) x_i)]^2}{e^{2\eta}} + \text{const}.\]

Log-posterior: Sum of log-likelihood and log-prior densities.

Run MCMC (e.g., via random-walk MH or Stan’s NUTS sampler) and diagnose convergence.

Show code 
set.seed(42)

sigma_true <- 1.0

     xlim = c(-0.5, 7.5), ylim = c(-2, 9),
     main = "Linear Regression Data")

Now run the MCMC sampler (pseudocode; in practice, use Stan, JAGS, or a hand-coded sampler):

Show code 
# Log-likelihood for y_i ~ N(b0 + b1 * x_i, sigma^2)
log_likelihood <- function(b0, b1, sigma, y, x) {
  mu <- b0 + b1 * x
  sum(dnorm(y, mu, sigma, log = TRUE))
}

# Log-posterior (unnormalized)
log_posterior <- function(theta, data) {
  b0 <- theta[1]
  alpha <- theta[2]
  eta <- theta[3]

  b1 <- tan(alpha)
  sigma <- exp(eta)

  # Priors
  log_prior_b0 <- dnorm(b0, 0, 10, log = TRUE)
  log_prior_alpha <- 0  # Flat in alpha
  log_prior_eta <- 0    # Flat in eta (Jeffreys for sigma)

  # Likelihood
  log_like <- log_likelihood(b0, b1, sigma, data$y, data$x)

  log_like + log_prior_b0 + log_prior_alpha + log_prior_eta
}

After running MCMC for 50,000 iterations (with 10,000 burn-in), extract the posterior samples and compute diagnostics:

12.10 Nonlinear Models: Laplace Approximation and MCMC

When \(f(x; \theta)\) is nonlinear in \(\theta\), the posterior cannot be written in closed form. Two main approaches:

Laplace Approximation: Expand the log-posterior around the MAP estimate \(\hat{\boldsymbol{\theta}}\) as a multivariate Gaussian: \[p(\boldsymbol{\theta} \mid D) \approx \mathcal{N}(\hat{\boldsymbol{\theta}}, [-\nabla^2 \log p(\boldsymbol{\theta} \mid D) |_{\hat{\boldsymbol{\theta}}}]^{-1}).\]

This is fast and often accurate for well-behaved posteriors. For multimodal or heavy-tailed posteriors, it can fail.

MCMC: Sample directly from the posterior, handling arbitrary nonlinearity and non-Gaussian shapes. The disadvantage is computational cost; the advantage is generality and accuracy. Modern samplers like NUTS (in Stan) adapt the step size and use Hamiltonian dynamics, making MCMC scalable.

12.11 Posterior Predictive Checks

A fundamental tool in Bayesian regression is posterior predictive checking: generate new data under the posterior and compare to observed data.

For a regression model, the posterior predictive distribution is \[p(y_{\text{new}} \mid x_{\text{new}}, D) = \int p(y_{\text{new}} \mid x_{\text{new}}, \boldsymbol{\theta}) \, p(\boldsymbol{\theta} \mid D) \, d\boldsymbol{\theta}.\]

The marginal predictive distribution averages over parameter uncertainty, automatically accounting for both the fitted model and the noise in the data.

Posterior predictive checks: 1. For each posterior sample \(\boldsymbol{\theta}^{(k)}\), generate \(y_{\text{rep}}^{(k)}(x_i) \sim p(y \mid x_i, \boldsymbol{\theta}^{(k)})\) at each observed \(x_i\). 2. Compare the distribution of simulated \(y_{\text{rep}}\) to observed \(y\) using summary statistics (e.g., mean, variance, quantiles, or max absolute residual). 3. If the posterior predictive distribution is consistent with the observed data, the model is plausible.

12.12 Robustness and Extensions

12.12.1 Student-\(t\) Noise for Robustness

Gaussian noise is sensitive to outliers. A robust alternative is Student-\(t\) noise: \[y_i \sim t_\nu(f(x_i; \theta), \sigma^2),\] where \(\nu\) controls tail heaviness (small \(\nu\) allows larger outliers). As \(\nu \to \infty\), recovers Gaussian. Often set \(\nu = 3\) or \(\nu = 4\) for robustness.

The likelihood becomes \[p(y_i \mid \ldots) \propto \left[ 1 + \frac{[y_i - f(x_i; \theta)]^2}{\nu \sigma^2} \right]^{-(\nu+1)/2},\] which is heavier-tailed than Gaussian. MCMC is needed to fit such models.

12.12.2 Errors in All Variables

If both \(x_i\) and \(y_i\) are measured with error, the problem becomes significantly harder: \[x_i = x_i^{\text{true}} + \delta_{x,i}, \quad y_i = f(x_i^{\text{true}}; \theta) + \delta_{y,i}.\]

The true \(x_i^{\text{true}}\) must be treated as latent variables, increasing dimensionality. Bayesian approaches handle this naturally via hierarchical modeling, but require additional priors and tuning.

12.13 Summary

Bayesian regression provides a principled framework for learning functional relationships:

  • Linear models (parameters enter linearly) admit conjugate closed-form posteriors via Gaussian theory, enabling exact inference and efficient prediction.
  • Posterior prediction integrates over parameter uncertainty, yielding predictive distributions that account for both fitted function and observation noise.
  • Nonlinear and generalized models (nonlinear parameters, non-Gaussian likelihoods) require MCMC sampling or approximations.
  • Practical MCMC workflow (specify model, choose priors, write log-posterior, run sampler, diagnose, summarize, plot) is systematic and reproducible.
  • Posterior predictive checks verify model adequacy and reveal misfit.
  • Robustness (heavy-tailed noise) and latent variable models (measurement error) extend the framework to real data.

The same logic applies across simple linear regression, polynomial fits, nonlinear curves, and generalized linear models: only the parameterization and likelihood change; the Bayesian template—prior \(\times\) likelihood $/ $ evidence—remains constant.

## Introduction

Regression analysis seeks to learn a functional relationship between input variables $x$ and observed response variables $y$ from data. In the Bayesian framework, we construct a probabilistic model that combines a generative (noise-free) function $f(x; \theta)$ with a noise model, then use Bayes' theorem to infer the posterior distribution $p(\theta \mid D)$ of the parameters. For regression models where the parameters enter linearly, exact posterior inference is tractable via conjugate analysis. For nonlinear models or non-Gaussian likelihoods, we rely on Markov chain Monte Carlo (MCMC) sampling. This chapter develops both the theory and practice of Bayesian regression: from the closed-form solution for linear Gaussian regression to the complete MCMC workflow for fitting arbitrary nonlinear regression models and making probabilistic predictions.

## Regression Model: Signal Plus Noise

### Model Formulation

The basic regression model specifies that each observed response $y_i$ arises from a noise-free mean function plus random noise:

$$y_i = f(x_i; \theta) + \varepsilon_i, \quad \varepsilon_i \sim \mathcal{N}(0, \sigma^2).$$

Here:
- $x_i$ is the regressor (input, explanatory variable), assumed fixed and measured without error.
- $y_i$ is the response (dependent, outcome variable).
- $f(x_i; \theta)$ is the signal—the deterministic part.
- $\varepsilon_i$ is measurement noise or intrinsic scatter, i.i.d. Gaussian with variance $\sigma^2$.

### Likelihood for Gaussian Noise

Under independent Gaussian noise, each observation has conditional distribution
$$p(y_i \mid x_i, \theta) = \mathcal{N}(f(x_i; \theta), \sigma^2).$$

For $N$ independent observations $D = \{(x_i, y_i)\}_{i=1}^N$, the joint likelihood factorizes:
$$p(D \mid \theta) = \prod_{i=1}^N p(y_i \mid x_i, \theta) = \prod_{i=1}^N \mathcal{N}(f(x_i; \theta), \sigma^2).$$

Taking logarithms,
$$\log p(D \mid \theta) = -N \log\sigma - \frac{1}{2}\log(2\pi) - \frac{1}{2\sigma^2} \sum_{i=1}^N [y_i - f(x_i; \theta)]^2 + \text{const}.$$

## Linear Models and the Design Matrix

### Linearity in Parameters

A regression model is **linear** if the parameters enter linearly, even if the features depend nonlinearly on the inputs:

$$y_i = \boldsymbol{\phi}(x_i)^\top \mathbf{w} + \varepsilon_i,$$

where $\boldsymbol{\phi}(x) = [\phi_1(x), \ldots, \phi_M(x)]^\top$ is a feature map and $\mathbf{w} = [w_0, \ldots, w_{M-1}]^\top$ is the weight vector.

:::{.definition}
**Examples of linear regression models:**
- Straight line: $\boldsymbol{\phi}(x) = (1, x)$, $f(x) = w_0 + w_1 x$.
- Polynomial: $\boldsymbol{\phi}(x) = (1, x, x^2, \ldots, x^{M-1})$, $f(x) = \sum_{j=0}^{M-1} w_j x^j$.
- Fourier: $\boldsymbol{\phi}(x) = (1, \sin(x), \cos(x), \ldots)$.
- Linear in unknown parameters: If $f(x) = w_0 e^{-\lambda x}$ with $\lambda$ known, it is linear in $w_0$.
- **Not** linear in unknown parameters: $f(x) = w_0 e^{-\lambda x}$ if $\lambda$ is unknown (exponential, logistic, etc.).
:::

### Design Matrix Notation

For $N$ data points and $M$ basis functions, define the $N \times M$ **design matrix** (or regressor matrix)
$$\boldsymbol{\Phi} = \begin{bmatrix}
\phi_1(x_1) & \phi_2(x_1) & \cdots & \phi_M(x_1) \\
\phi_1(x_2) & \phi_2(x_2) & \cdots & \phi_M(x_2) \\
\vdots & \vdots & \ddots & \vdots \\
\phi_1(x_N) & \phi_2(x_N) & \cdots & \phi_M(x_N)
\end{bmatrix}.$$

The linear model becomes
$$\mathbf{y} = \boldsymbol{\Phi} \mathbf{w} + \boldsymbol{\varepsilon}, \quad \boldsymbol{\varepsilon} \sim \mathcal{N}(\mathbf{0}, \sigma^2 \mathbf{I}_N),$$

and the likelihood is
$$p(\mathbf{y} \mid \boldsymbol{\Phi}, \mathbf{w}, \sigma^2) = \mathcal{N}(\mathbf{y} \mid \boldsymbol{\Phi} \mathbf{w}, \sigma^2 \mathbf{I}_N).$$

## Bayesian Linear Regression: Closed-Form Solution

### Conjugate Prior and Posterior

Assume $\sigma^2$ is known. Choose a Gaussian prior on $\mathbf{w}$:
$$p(\mathbf{w}) = \mathcal{N}(\mathbf{w} \mid \mathbf{m}_0, \mathbf{S}_0).$$

The likelihood is Gaussian in $\mathbf{w}$; by conjugacy, the posterior is also Gaussian:

:::{.theorem}
**Bayesian Linear Regression with Gaussian Prior:**

Given likelihood $p(\mathbf{y} \mid \boldsymbol{\Phi}, \mathbf{w}, \sigma^2) = \mathcal{N}(\mathbf{y} \mid \boldsymbol{\Phi} \mathbf{w}, \sigma^2 \mathbf{I}_N)$ and prior $\mathcal{N}(\mathbf{w} \mid \mathbf{m}_0, \mathbf{S}_0)$, the posterior is
$$p(\mathbf{w} \mid D, \sigma^2) = \mathcal{N}(\mathbf{w} \mid \mathbf{m}_N, \mathbf{S}_N),$$

where
\begin{align}
\mathbf{S}_N^{-1} &= \mathbf{S}_0^{-1} + \sigma^{-2} \boldsymbol{\Phi}^\top \boldsymbol{\Phi}, \\
\mathbf{m}_N &= \mathbf{S}_N \left( \mathbf{S}_0^{-1} \mathbf{m}_0 + \sigma^{-2} \boldsymbol{\Phi}^\top \mathbf{y} \right).
\end{align}
:::

**Interpretation:** Precisions add. The posterior mean is a precision-weighted average of the prior mean and the data-driven estimate. The posterior covariance $\mathbf{S}_N$ shrinks relative to $\mathbf{S}_0$.

### Special Case: Weak Isotropic Prior

A common choice is the weak prior $\mathbf{m}_0 = \mathbf{0}$ and $\mathbf{S}_0 = \sigma_0^2 \mathbf{I}_M$:
$$\mathbf{S}_N^{-1} = \frac{1}{\sigma_0^2} \mathbf{I}_M + \sigma^{-2} \boldsymbol{\Phi}^\top \boldsymbol{\Phi},$$
$$\mathbf{m}_N = \sigma^{-2} \mathbf{S}_N \boldsymbol{\Phi}^\top \mathbf{y}.$$

### Connection to Ridge Regression

Ordinary least squares minimizes $\|\mathbf{y} - \boldsymbol{\Phi} \mathbf{w}\|^2$. Ridge regression adds an L2 penalty:
$$L_{\text{ridge}} = \|\mathbf{y} - \boldsymbol{\Phi} \mathbf{w}\|^2 + \lambda \|\mathbf{w}\|^2.$$

The ridge estimator is $\hat{\mathbf{w}}_{\text{ridge}} = (\boldsymbol{\Phi}^\top \boldsymbol{\Phi} + \lambda \mathbf{I})^{-1} \boldsymbol{\Phi}^\top \mathbf{y}$.

Ridge regression is **maximum a posteriori (MAP) estimation** under a Gaussian prior: the prior variance relates to the penalty by $\lambda = \sigma^2 / \sigma_0^2$. The Bayesian approach goes further by quantifying posterior uncertainty via the full covariance $\mathbf{S}_N$.

## Posterior Predictive Distribution

### Parameter Uncertainty and Prediction Uncertainty

For a new input $x_* $, we wish to predict $y_*$. The posterior predictive distribution marginalizes over parameter uncertainty:
$$p(y_* \mid x_*, D, \sigma^2) = \int p(y_* \mid x_*, \mathbf{w}, \sigma^2) \, p(\mathbf{w} \mid D, \sigma^2) \, d\mathbf{w}.$$

Since both terms are Gaussian, the integral is tractable:

:::{.theorem}
**Posterior Predictive Distribution:**

Let $\boldsymbol{\phi}_* = \boldsymbol{\phi}(x_*)$. Then
$$p(y_* \mid x_*, D, \sigma^2) = \mathcal{N}\left( y_* \,\big|\, \boldsymbol{\phi}_*^\top \mathbf{m}_N, \, \sigma^2 + \boldsymbol{\phi}_*^\top \mathbf{S}_N \boldsymbol{\phi}_* \right).$$
:::

**Two sources of uncertainty in prediction:**
1. **Observation noise:** $\sigma^2$ (irreducible error in $y_*$).
2. **Parameter uncertainty:** $\boldsymbol{\phi}_*^\top \mathbf{S}_N \boldsymbol{\phi}_*$ (uncertainty in the regression function $f(x_*)$).

The posterior predictive band widens away from the data due to increasing parameter uncertainty.

## Unknown Noise Level

In practice, $\sigma^2$ is usually unknown and inferred from data. The parameters become $\boldsymbol{\theta} = (\mathbf{w}, \sigma)$, and we use a prior on the noise scale. A standard choice is the **Jeffreys prior** for scale parameters:
$$p(\sigma) \propto \frac{1}{\sigma}, \quad \text{or equivalently} \quad p(\log \sigma) \text{ is uniform}.$$

The joint posterior is
$$p(\mathbf{w}, \sigma \mid D) \propto p(\mathbf{y} \mid \boldsymbol{\Phi}, \mathbf{w}, \sigma) \, p(\mathbf{w}) \, p(\sigma).$$

When both $\mathbf{w}$ and $\sigma$ are unknown, the posterior no longer has a closed form in closed-form distributions, but conjugate analysis can still apply to marginal posteriors or conditional distributions. For computational convenience and to make use of existing MCMC software, we often resort to sampling.

## Nonlinear Regression and MCMC

### Why Nonlinear Models Require MCMC

A model is **nonlinear in parameters** if $f(x; \theta)$ is a nonlinear function of some $\theta$. Examples:
- Exponential: $f(x) = a e^{b x}$ (nonlinear in both $a$ and $b$ unless one is known).
- Logistic: $f(x) = \frac{L}{1 + e^{-k(x - x_0)}}$ (nonlinear in location $x_0$, scale $k$, amplitude $L$).
- Power law: $f(x) = a x^b$ (nonlinear in exponent $b$).

For such models, the likelihood is no longer Gaussian in the parameters, and conjugate priors do not yield closed-form posteriors. Similarly, generalized linear models (logistic regression for binary data, Poisson regression for counts) have non-Gaussian likelihoods, precluding analytical solutions.

In these cases, we must either:
- Use **approximations** (Laplace, variational inference).
- Use **MCMC** or other sampling methods to draw from $p(\boldsymbol{\theta} \mid D) \propto p(D \mid \boldsymbol{\theta}) \, p(\boldsymbol{\theta})$.

## Practical MCMC Fitting Workflow

The following seven-step workflow is the standard approach for fitting regression models via MCMC:

### Step 1: Specify the Generative Model and Noise Model

Write down:
- The regression function: $f(x; \boldsymbol{\theta})$.
- The noise distribution: e.g., $\varepsilon \sim \mathcal{N}(0, \sigma^2)$.
- The observation model: $y_i \sim \mathcal{N}(f(x_i; \boldsymbol{\theta}), \sigma^2)$.

### Step 2: Choose Priors for All Parameters

Specify weakly informative priors for each parameter. For regression coefficients, use broad Gaussians (e.g., $\mathcal{N}(0, 10^2)$). For scale parameters like $\sigma$, use a Jeffreys prior $p(\sigma) \propto 1/\sigma$ or truncated exponentials. To avoid boundary issues, sample on the log-scale: $\eta = \log \sigma$.

:::{.note}
**Reparameterization for Bounded Parameters:**

Parameters constrained to positive values (like $\sigma > 0$) should be sampled in transformed space:
- Replace $\sigma$ with $\eta = \log \sigma$, so $\sigma = e^\eta$.
- A flat prior on $\eta$ corresponds to a Jeffreys prior on $\sigma$.
- The Jacobian $|d\sigma / d\eta| = e^\eta = \sigma$ automatically cancels the $1/\sigma$ prior, yielding a uniform prior in log-space.

For slope parameters, reparameterize via angle: $\beta = \tan(\alpha)$, so all slopes are finite and comparable.
:::

### Step 3: Write Log-Prior, Log-Likelihood, and Log-Posterior Functions

Define functions that compute:
\begin{align}
\log p(\boldsymbol{\theta}) &: \text{ sum of log-prior densities for all parameters}, \\
\log p(D \mid \boldsymbol{\theta}) &: \text{ log-likelihood, typically } \sum_{i=1}^N \log p(y_i \mid x_i, \boldsymbol{\theta}), \\
\log p(\boldsymbol{\theta} \mid D) &= \log p(D \mid \boldsymbol{\theta}) + \log p(\boldsymbol{\theta}) + \text{const}.
\end{align}

For MCMC, we only need the unnormalized posterior (up to an additive constant).

### Step 4: Run MCMC and Tune Proposal Covariance

Initialize parameters at a reasonable starting point (e.g., a classical fit or MAP estimate). Specify a proposal distribution, typically a multivariate Gaussian with tuned covariance. Run MCMC (e.g., random-walk Metropolis-Hastings, or preferably NUTS via Stan) for a burn-in period, then for additional sampling iterations.

Tune the proposal variance to achieve a target acceptance rate (typically 20--30% for random-walk MH, automatic for NUTS).

### Step 5: Diagnose and Inspect the Chain

:::{.remark}
**Key Diagnostics:**
- **Trace plots:** Plot $\theta_j$ versus iteration. Traces should resemble white noise (good mixing), not slow drifts or stuck chains.
- **Autocorrelation function (ACF):** Compute $\text{Corr}(\theta_j^{(k)}, \theta_j^{(k+\ell)})$ as a function of lag $\ell$. Rapid decay (small effective sample size if slow decay) indicates good mixing.
- **Effective sample size (ESS):** $\text{ESS} = N_{\text{samples}} / (1 + 2 \sum_{\ell=1}^\infty \text{Corr}(\ell))$. Rule of thumb: aim for ESS $> 1000$ per parameter.
- **$\hat{R}$ (Gelman-Rubin statistic):** Ratio of between-chain to within-chain variance. $\hat{R} < 1.01$ indicates convergence; $\hat{R} > 1.05$ indicates poor mixing or insufficient burn-in.
- **Joint posterior pairs plot:** Scatter plot of all pairs of parameters. Reveals correlations and identifies poorly identified parameters.
:::

### Step 6: Summarize Parameters

Compute posterior summaries from the converged samples (after thinning to reduce autocorrelation if desired):
- **Maximum a posteriori (MAP):** The sample with highest posterior density.
- **Posterior mean:** Average of all samples.
- **Posterior covariance:** Empirical covariance of samples.
- **Credible intervals:** 2.5th and 97.5th quantiles (or other quantiles) per parameter.

Always transform samples back to the original parameterization before computing summaries.

### Step 7: Plot Predictions and Posterior Predictive Checks

1. **Data plot with error bars:** Show observed $(x_i, y_i)$ with vertical error bars at $\pm \sigma_{\text{MAP}}$ or $\pm$ posterior SD of $y_i$.

2. **Fitted regression curve:** Plot the posterior mean fit $\hat{f}(x) = \boldsymbol{\phi}(x)^\top \mathbb{E}[\mathbf{w} \mid D]$.

3. **Credible band for the mean curve:** For a grid of $x$ values, compute the posterior quantiles of $f(x; \boldsymbol{\theta})$ by evaluating the function at each posterior sample. This shows uncertainty in the regression function, not including observation noise.

4. **Posterior predictive band:** For future observations, include both parameter uncertainty and noise. For each $x$ on the grid and each posterior sample $\boldsymbol{\theta}^{(k)}$, generate $y^{(k)} \sim \mathcal{N}(f(x; \boldsymbol{\theta}^{(k)}), \sigma^{(k)2})$. The 2.5th and 97.5th quantiles across samples form the prediction interval.

## Example: Linear Regression with MCMC

Consider fitting a straight line $f(x) = b_0 + b_1 x$ to observed data $\{(x_i, y_i)\}$, with unknown noise $\sigma$.

**Parameters (transformed):** $\boldsymbol{\theta} = (b_0, \alpha, \eta)$ where $b_1 = \tan(\alpha)$ and $\eta = \log \sigma$.

**Priors:**
- $b_0 \sim \mathcal{N}(0, 10^2)$.
- $\alpha \sim \text{Uniform}(-\pi/2, \pi/2)$ (equivalently, $p(\alpha) \propto 1$).
- $\eta \sim \text{Uniform}$ (flat prior on $\log \sigma$, Jeffreys prior on $\sigma$).

**Log-likelihood:**
$$\log p(\mathbf{y} \mid b_0, \alpha, \eta) = -N \eta - \frac{1}{2} \sum_{i=1}^N \frac{[y_i - (b_0 + \tan(\alpha) x_i)]^2}{e^{2\eta}} + \text{const}.$$

**Log-posterior:** Sum of log-likelihood and log-prior densities.

Run MCMC (e.g., via random-walk MH or Stan's NUTS sampler) and diagnose convergence.

```{r}
#| label: lst-linear-data

#| fig-cap: "Linear regression example: 8 data points from y = 0 + x + noise."

set.seed(42)
N <- 8
x <- seq(0, 7, length.out = N)
sigma_true <- 1.0
y <- 0 + 1 * x + rnorm(N, 0, sigma_true)

plot(x, y, pch = 19, col = "black", xlab = "x", ylab = "y",
     xlim = c(-0.5, 7.5), ylim = c(-2, 9),
     main = "Linear Regression Data")
```

Now run the MCMC sampler (pseudocode; in practice, use Stan, JAGS, or a hand-coded sampler):

```{r}
#| label: lst-linear-mcmc-functions
#| lst-cap: "Log-posterior for linear regression."

# Log-likelihood for y_i ~ N(b0 + b1 * x_i, sigma^2)
log_likelihood <- function(b0, b1, sigma, y, x) {
  mu <- b0 + b1 * x
  sum(dnorm(y, mu, sigma, log = TRUE))
}

# Log-posterior (unnormalized)
log_posterior <- function(theta, data) {
  b0 <- theta[1]
  alpha <- theta[2]
  eta <- theta[3]

  b1 <- tan(alpha)
  sigma <- exp(eta)

  # Priors
  log_prior_b0 <- dnorm(b0, 0, 10, log = TRUE)
  log_prior_alpha <- 0  # Flat in alpha
  log_prior_eta <- 0    # Flat in eta (Jeffreys for sigma)

  # Likelihood
  log_like <- log_likelihood(b0, b1, sigma, data$y, data$x)

  log_like + log_prior_b0 + log_prior_alpha + log_prior_eta
}
```

## Nonlinear Models: Laplace Approximation and MCMC

When $f(x; \theta)$ is nonlinear in $\theta$, the posterior cannot be written in closed form. Two main approaches:

**Laplace Approximation:** Expand the log-posterior around the MAP estimate $\hat{\boldsymbol{\theta}}$ as a multivariate Gaussian:
$$p(\boldsymbol{\theta} \mid D) \approx \mathcal{N}(\hat{\boldsymbol{\theta}}, [-\nabla^2 \log p(\boldsymbol{\theta} \mid D) |_{\hat{\boldsymbol{\theta}}}]^{-1}).$$

This is fast and often accurate for well-behaved posteriors. For multimodal or heavy-tailed posteriors, it can fail.

**MCMC:** Sample directly from the posterior, handling arbitrary nonlinearity and non-Gaussian shapes. The disadvantage is computational cost; the advantage is generality and accuracy. Modern samplers like NUTS (in Stan) adapt the step size and use Hamiltonian dynamics, making MCMC scalable.

## Posterior Predictive Checks

A fundamental tool in Bayesian regression is **posterior predictive checking**: generate new data under the posterior and compare to observed data.

:::{.definition}
For a regression model, the posterior predictive distribution is
$$p(y_{\text{new}} \mid x_{\text{new}}, D) = \int p(y_{\text{new}} \mid x_{\text{new}}, \boldsymbol{\theta}) \, p(\boldsymbol{\theta} \mid D) \, d\boldsymbol{\theta}.$$

The marginal predictive distribution averages over parameter uncertainty, automatically accounting for both the fitted model and the noise in the data.

Posterior predictive checks:
1. For each posterior sample $\boldsymbol{\theta}^{(k)}$, generate $y_{\text{rep}}^{(k)}(x_i) \sim p(y \mid x_i, \boldsymbol{\theta}^{(k)})$ at each observed $x_i$.
2. Compare the distribution of simulated $y_{\text{rep}}$ to observed $y$ using summary statistics (e.g., mean, variance, quantiles, or max absolute residual).
3. If the posterior predictive distribution is consistent with the observed data, the model is plausible.
:::

## Robustness and Extensions

### Student-$t$ Noise for Robustness

Gaussian noise is sensitive to outliers. A robust alternative is **Student-$t$ noise**:
$$y_i \sim t_\nu(f(x_i; \theta), \sigma^2),$$
where $\nu$ controls tail heaviness (small $\nu$ allows larger outliers). As $\nu \to \infty$, recovers Gaussian. Often set $\nu = 3$ or $\nu = 4$ for robustness.

The likelihood becomes
$$p(y_i \mid \ldots) \propto \left[ 1 + \frac{[y_i - f(x_i; \theta)]^2}{\nu \sigma^2} \right]^{-(\nu+1)/2},$$
which is heavier-tailed than Gaussian. MCMC is needed to fit such models.

### Errors in All Variables

If both $x_i$ and $y_i$ are measured with error, the problem becomes significantly harder:
$$x_i = x_i^{\text{true}} + \delta_{x,i}, \quad y_i = f(x_i^{\text{true}}; \theta) + \delta_{y,i}.$$

The true $x_i^{\text{true}}$ must be treated as latent variables, increasing dimensionality. Bayesian approaches handle this naturally via hierarchical modeling, but require additional priors and tuning.

## Summary

Bayesian regression provides a principled framework for learning functional relationships:

- **Linear models** (parameters enter linearly) admit conjugate closed-form posteriors via Gaussian theory, enabling exact inference and efficient prediction.
- **Posterior prediction** integrates over parameter uncertainty, yielding predictive distributions that account for both fitted function and observation noise.
- **Nonlinear and generalized models** (nonlinear parameters, non-Gaussian likelihoods) require MCMC sampling or approximations.
- **Practical MCMC workflow** (specify model, choose priors, write log-posterior, run sampler, diagnose, summarize, plot) is systematic and reproducible.
- **Posterior predictive checks** verify model adequacy and reveal misfit.
- **Robustness** (heavy-tailed noise) and **latent variable models** (measurement error) extend the framework to real data.

The same logic applies across simple linear regression, polynomial fits, nonlinear curves, and generalized linear models: only the parameterization and likelihood change; the Bayesian template—prior $\times$ likelihood $/ $ evidence—remains constant.
