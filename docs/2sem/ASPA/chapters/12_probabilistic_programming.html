13  Probabilistic programming: JAGS, Stan, ADVI

13.1 Introduction: from hand-coded to declarative models

In preceding chapters, we implemented Markov chain Monte Carlo by hand: we wrote log-likelihood and log-prior functions, tuned proposal covariance matrices, diagnosed convergence, and iterated. This approach is powerful pedagogically—it reveals exactly what is being sampled and how—but it is labour-intensive and error-prone for realistic models.

Real-world inference often requires many hyperparameters, hierarchical structure, and thousands of latent variables. Writing the full log-posterior by hand for such models is impractical. We need a language in which to declare the model in terms of its statistical structure, and delegate the numerical sampling to software.

Probabilistic programming languages (PPLs) solve this. They allow you to write: \[\text{model + data} \to p(\theta \mid \mathbf{y}).\]

Two mainstream PPLs in the applied Bayesian community are JAGS and Stan. We also cover variational inference, an alternative to MCMC that solves the posterior by optimisation, and its automation in ADVI (automatic differentiation variational inference).

13.2 The classical limitation: conjugacy

In the first chapters of this course, we exploited conjugate priors: when the prior and likelihood share a specific functional form, the posterior is tractable analytically. For example, a Beta prior on a Bernoulli probability yields a Beta posterior; a Normal prior on a Gaussian mean (with known variance) yields a Normal posterior.

Conjugacy is rare in practice. Most real models are non-conjugate: the posterior is not available in closed form. In such cases we have two paths:

  1. MCMC: Sample from the posterior.
  2. Variational inference: Approximate the posterior with a tractable family.

Both JAGS and Stan use MCMC. JAGS uses Gibbs sampling or Metropolis-within-Gibbs (no gradient information). Stan uses Hamiltonian Monte Carlo with automatic differentiation (gradient-based), which mixes much faster.

13.3 JAGS: Gibbs sampling with a model syntax

13.3.1 Overview

JAGS stands for Just Another Gibbs Sampler. It is a language and engine for specifying Bayesian hierarchical models and sampling from them using Gibbs updates or Metropolis-Hastings steps.

The key idea: you write the model in a declarative syntax that lists the conditional distribution of each variable given its parents. JAGS parses this, determines which variables can be sampled by Gibbs and which need Metropolis-Hastings, and executes the sampler automatically.

13.3.2 Model syntax

A JAGS model comprises three optional blocks: data, model, and inits.

The data block declares observed quantities (fixed).

The model block specifies the generative structure: for each stochastic node (parameter, latent variable, observation), you write its conditional distribution.

The inits block (usually separate in R) provides starting values.

Example: centered quadratic regression.

model {
  # Likelihood
  for (i in 1:N) {
    y[i] ~ dnorm(mu[i], tau)
    mu[i] <- a0 + a1 * z[i] + a2 * pow(z[i], 2)
  }

  # Priors
  a0 ~ dnorm(0, 1/100)
  a1 ~ dnorm(0, 1/100)
  a2 ~ dnorm(0, 1/100)

  # Precision parameterization
  tau <- 1 / pow(sigma, 2)
  sigma ~ dunif(0, 20)
}

Key syntax notes:

  • JAGS uses precision \(\tau = 1/\sigma^2\), not standard deviation.
  • Deterministic transformations use <-; stochastic nodes use ~.
  • Distributions are prefixed d: dnorm, dunif, dbinom, etc.
  • The prior on \(\sigma\) must be proper.

13.3.3 Workflow in R with rjags

library(rjags)
library(coda)

# Prepare data
jags_data <- list(
  N = nrow(obsdata),
  z = obsdata$z,
  y = obsdata$y
)

# Initialize
inits <- function() list(a0=0, a1=0, a2=0, sigma=1)

# Compile
jm <- jags.model(textConnection(model_string),
                 data=jags_data,
                 inits=inits,
                 n.chains=4)

# Adapt (burn-in)
update(jm, 2000)

# Sample
samp <- coda.samples(jm,
                     variable.names=c("a0","a1","a2","sigma"),
                     n.iter=20000)

# Summarize
summary(samp)
effectiveSize(samp)

Advantages of JAGS:

  • Simple declarative syntax.
  • Automatic Gibbs / MH switching.
  • Handles discrete and continuous parameters.

Disadvantages:

  • Gibbs sampling mixes slowly for correlated posteriors.
  • No gradient information; cannot exploit manifold geometry.
  • Precision parameterization less intuitive than standard deviation.

13.4 Stan: Hamiltonian Monte Carlo with gradients

13.4.1 Overview

Stan is a probabilistic programming language that uses Hamiltonian Monte Carlo with the No-U-Turn Sampler (NUTS) algorithm. By leveraging automatic differentiation to compute gradients of the log-posterior, Stan explores the posterior much more efficiently than Gibbs sampling, especially in high dimensions.

Modern Stan workflows use cmdstanr, a lightweight R interface.

13.4.2 Model structure

A Stan program has five optional blocks, executed in order:

  1. data: Observed quantities, fixed.
  2. transformed data: Derived quantities computed once.
  3. parameters: The target of inference.
  4. transformed parameters: Deterministic functions of parameters.
  5. model: The log-posterior (likelihood + priors).
  6. generated quantities: Posterior predictions.

13.4.3 Example: centered quadratic regression

data {
  int<lower=1> N;
  vector[N] z;
  vector[N] y;
}

parameters {
  real a0;
  real a1;
  real a2;
  real<lower=0> sigma;
}

model {
  // Likelihood
  y ~ normal(a0 + a1 * z + a2 * square(z), sigma);

  // Priors
  a0 ~ normal(0, 20);
  a1 ~ normal(0, 20);
  a2 ~ normal(0, 20);
  sigma ~ normal(0, 10);  // Half-normal (due to lower=0)
}

Key syntax notes:

  • Type declarations: real, vector[N], int.
  • Constraints are declared inline: <lower=0>, <upper=1>.
  • Stan samples unconstrained internally; constraints are handled via smooth transformations (e.g., log for positive parameters).
  • Standard deviation is natural; no precision.
  • Vectorized operations (y ~ normal(...) broadcasts over the vector) are encouraged.

13.4.4 Workflow in R with cmdstanr

library(cmdstanr)

# Prepare data
stan_data <- list(
  N = nrow(obsdata),
  z = obsdata$z,
  y = obsdata$y
)

# Compile
mod <- cmdstan_model("quad_centered.stan")

# Sample
fit <- mod$sample(
  data = stan_data,
  chains = 4,
  parallel_chains = 4,
  iter_warmup = 1000,
  iter_sampling = 2000,
  seed = 123
)

# Summary
fit$summary(c("a0", "a1", "a2", "sigma"))

# Diagnostics
fit$cmdstan_diagnose()

13.4.5 Posterior diagnostics in Stan

Stan reports several convergence statistics:

  • \(\hat{R}\) (potential scale reduction factor): Should be \(< 1.01\). Values \(> 1.05\) indicate poor convergence.
  • ESS (effective sample size): Bulk ESS and tail ESS. Rule of thumb: both \(\gtrsim 400\) per chain. Low ESS indicates high autocorrelation.
  • Divergent transitions: Occur when the leapfrog integrator leaves the high-probability region. May indicate a multimodal or heavy-tailed target. Typical remedy: increase adapt_delta (default 0.8) to 0.9 or 0.95.

13.6 Variational inference: approximate posterior via optimisation

13.6.1 The KL divergence decomposition

MCMC samples exactly (asymptotically) from the posterior \(p(\theta \mid \mathbf{y})\). Variational inference instead optimises over a family of tractable distributions \(\mathcal{Q}\) to find the member \(q^*(\theta)\) that best approximates the posterior.

Define the Kullback–Leibler divergence: \[\mathrm{KL}(q \parallel p) = \mathbb{E}_q \left[ \log \frac{q(\theta)}{p(\theta \mid \mathbf{y})} \right].\]

By the law of total probability, \[\log p(\mathbf{y}) = \mathbb{E}_q [ \log p(\mathbf{y}, \theta) - \log q(\theta) ] + \mathrm{KL}(q \parallel p),\]

which rearranges to

\[\boxed{\log p(\mathbf{y}) = \mathcal{L}(q) + \mathrm{KL}(q \parallel p)}.\]

Here \(\mathcal{L}(q)\) is the evidence lower bound (ELBO): \[\mathcal{L}(q) = \mathbb{E}_q [ \log p(\mathbf{y}, \theta) - \log q(\theta) ].\]

Since \(\log p(\mathbf{y})\) is fixed and \(\mathrm{KL}(q \parallel p) \geq 0\) always, maximising the ELBO minimises the KL divergence. Thus the optimal approximation is

\[q^*(\theta) = \arg\max_{q \in \mathcal{Q}} \mathcal{L}(q).\]

13.6.2 Mean-field approximation

To make the optimisation tractable, we restrict \(\mathcal{Q}\) to a mean-field family: \[q(\theta) = \prod_{i=1}^{d} q_i(\theta_i),\]

i.e., a product of independent marginal distributions (one per parameter). This destroys posterior correlations but is far easier to optimise than a full covariance Gaussian.

For each parameter, \(q_i\) is typically a standard distribution (Gaussian, Beta, Gamma, etc.).

13.6.3 The reparameterisation trick

To optimise the ELBO by stochastic gradient descent, we need to differentiate through expectations. The reparameterisation trick rewrites an expectation: \[\mathbb{E}_{q(\theta)} [f(\theta)] = \mathbb{E}_{\epsilon \sim N(0, I)} [f(\mu + \sigma \epsilon)],\]

where \(\mu\) and \(\sigma\) are the (optimisable) parameters of \(q\), and \(\epsilon\) is auxiliary randomness. Now we can differentiate w.r.t. \(\mu\) and \(\sigma\) using the chain rule.

13.7 ADVI: Automatic Differentiation Variational Inference

13.7.1 Automatic transformation to unconstrained space

Many parameters have constraints: \(\sigma > 0\), \(p \in [0, 1]\), etc. ADVI automatically transforms such parameters to an unconstrained space, optimises a Gaussian approximation there, and back-transforms samples.

For example, \(\sigma > 0\) is mapped to \(\eta = \log \sigma \in \mathbb{R}\). The variational family becomes \(q(\eta)\) (a univariate Gaussian in \(\eta\)), and we sample \(\eta \sim N(\mu, \sigma^2)\), then set \(\sigma = \exp(\eta)\).

13.7.2 Gaussian variational family

ADVI uses a multivariate Gaussian approximation: \[q(\theta) = N(\theta \mid \mu, \Sigma),\]

where \(\Sigma\) is either diagonal (mean-field) or full-rank. The parameters \((\mu, \Sigma)\) are optimised by stochastic gradient ascent on the ELBO.

13.7.3 Stan’s vb() interface

# Variational inference in Stan (instead of MCMC)
fit_vb <- mod$variational(
  data = stan_data,
  algorithm = "meanfield",  # or "fullrank"
  iter = 10000,
  seed = 123
)

# Approximately posterior samples
draws <- fit_vb$draws()

Advantages of ADVI:

  • Much faster than MCMC for high-dimensional models.
  • Deterministic; no convergence diagnostics needed.
  • Useful for exploratory analysis.

Disadvantages:

  • Mean-field assumption often too restrictive; underestimates posterior variance.
  • No automatic diagnostic (unlike MCMC diagnostics like \(\hat{R}\)).
  • For posterior inference, MCMC is often preferable.

13.8 Practical workflow: from hand-rolled MCMC to PPLs

The progression we have traced reflects a real scientific workflow:

  1. Prototype with hand-coded MCMC (Chapter 11 onwards): Learn the model, understand the geometry of the posterior, diagnose what works and what doesn’t.

  2. Apply parametrisation tricks: Center and scale predictors, use whitening transforms, exploit posterior covariance structure.

  3. Upgrade to a gradient-based sampler: Use HMC or NUTS to exploit smoothness of the log-posterior.

  4. Automate with Stan or JAGS: Declare the model once, let the software handle sampling, diagnostics, and adaptation.

  5. Consider variational inference if:

    • You need very fast approximate inference for exploration.
    • Your model is too large for MCMC.
    • You are happy accepting mean-field correlations.

13.9 Failure modes and remedies

13.9.1 Multimodal posteriors

If the target distribution has multiple distinct modes, MCMC may get stuck in one mode and miss others. JAGS and Stan both struggle with multimodality.

Remedies:

  • Use parallel tempering (Chapter 10).
  • Run many short chains from dispersed starting values.
  • Reparametrise to reduce multimodality (e.g., via identifiability constraints).

13.9.2 Discrete parameters

Stan requires all parameters to be continuous. If your model includes discrete latent variables (e.g., mixture component labels), you must marginalise them out analytically (integrate over all possible values) before passing the model to Stan.

JAGS handles discrete nodes naturally.

13.9.3 Heavy-tailed posteriors

If the posterior has heavy tails (e.g., a mixture of normals with small weights), the leapfrog integrator in HMC can take very long steps and diverge. Divergent transitions are a warning.

Remedies:

  • Increase adapt_delta (0.8 → 0.95 or higher).
  • Increase max_treedepth.
  • Reparametrise to reduce tail heaviness (e.g., use a Student-\(t\) prior to allow more dispersed posteriors).

13.9.4 Weak identifiability

If two parameters affect the likelihood in nearly the same way, the posterior is highly correlated or even degenerate. MCMC mixes slowly; variational inference underestimates uncertainty.

Remedy:

  • Include informative priors that break the symmetry.
  • Reparametrise (e.g., use a sum-of-effects rather than separate effects).

13.10 What to report

For any Bayesian analysis using JAGS or Stan, report:

  1. Model specification: Likelihood, priors, parametrisation.
  2. Sampling method: Algorithm, number of chains, iterations, warmup.
  3. Convergence diagnostics: Trace plots, \(\hat{R}\), ESS, acceptance rate, divergent transitions (Stan only).
  4. Posterior summaries: Mean or median, 95% credible intervals.
  5. Posterior predictions: Regression curves with credible intervals; posterior predictive intervals if relevant.

Do not report precise posterior point estimates from a badly mixing chain—they are numerical artifacts, not inference.

## Variational inference: approximate posterior via optimisation

### The KL divergence decomposition

MCMC samples exactly (asymptotically) from the posterior $p(\theta \mid \mathbf{y})$. Variational inference instead *optimises* over a family of tractable distributions $\mathcal{Q}$ to find the member $q^*(\theta)$ that best approximates the posterior.

Define the Kullback–Leibler divergence:
$$\mathrm{KL}(q \parallel p) = \mathbb{E}_q \left[ \log \frac{q(\theta)}{p(\theta \mid \mathbf{y})} \right].$$

By the law of total probability,
$$\log p(\mathbf{y}) = \mathbb{E}_q [ \log p(\mathbf{y}, \theta) - \log q(\theta) ] + \mathrm{KL}(q \parallel p),$$

which rearranges to

$$\boxed{\log p(\mathbf{y}) = \mathcal{L}(q) + \mathrm{KL}(q \parallel p)}.$$

Here $\mathcal{L}(q)$ is the **evidence lower bound (ELBO)**:
$$\mathcal{L}(q) = \mathbb{E}_q [ \log p(\mathbf{y}, \theta) - \log q(\theta) ].$$

Since $\log p(\mathbf{y})$ is fixed and $\mathrm{KL}(q \parallel p) \geq 0$ always, *maximising the ELBO minimises the KL divergence*. Thus the optimal approximation is

$$q^*(\theta) = \arg\max_{q \in \mathcal{Q}} \mathcal{L}(q).$$

### Mean-field approximation

To make the optimisation tractable, we restrict $\mathcal{Q}$ to a *mean-field family*:
$$q(\theta) = \prod_{i=1}^{d} q_i(\theta_i),$$

i.e., a product of independent marginal distributions (one per parameter). This destroys posterior correlations but is far easier to optimise than a full covariance Gaussian.

For each parameter, $q_i$ is typically a standard distribution (Gaussian, Beta, Gamma, etc.).

### The reparameterisation trick

To optimise the ELBO by stochastic gradient descent, we need to differentiate through expectations. The **reparameterisation trick** rewrites an expectation:
$$\mathbb{E}_{q(\theta)} [f(\theta)] = \mathbb{E}_{\epsilon \sim N(0, I)} [f(\mu + \sigma \epsilon)],$$

where $\mu$ and $\sigma$ are the (optimisable) parameters of $q$, and $\epsilon$ is auxiliary randomness. Now we can differentiate w.r.t. $\mu$ and $\sigma$ using the chain rule.

## ADVI: Automatic Differentiation Variational Inference

### Automatic transformation to unconstrained space

Many parameters have constraints: $\sigma > 0$, $p \in [0, 1]$, etc. ADVI automatically transforms such parameters to an unconstrained space, optimises a Gaussian approximation there, and back-transforms samples.

For example, $\sigma > 0$ is mapped to $\eta = \log \sigma \in \mathbb{R}$. The variational family becomes $q(\eta)$ (a univariate Gaussian in $\eta$), and we sample $\eta \sim N(\mu, \sigma^2)$, then set $\sigma = \exp(\eta)$.

### Gaussian variational family

ADVI uses a multivariate Gaussian approximation:
$$q(\theta) = N(\theta \mid \mu, \Sigma),$$

where $\Sigma$ is either diagonal (mean-field) or full-rank. The parameters $(\mu, \Sigma)$ are optimised by stochastic gradient ascent on the ELBO.

### Stan's `vb()` interface

```r
# Variational inference in Stan (instead of MCMC)
fit_vb <- mod$variational(
  data = stan_data,
  algorithm = "meanfield",  # or "fullrank"
  iter = 10000,
  seed = 123
)

# Approximately posterior samples
draws <- fit_vb$draws()
```

**Advantages of ADVI:**

- Much faster than MCMC for high-dimensional models.
- Deterministic; no convergence diagnostics needed.
- Useful for exploratory analysis.

**Disadvantages:**

- Mean-field assumption often too restrictive; underestimates posterior variance.
- No automatic diagnostic (unlike MCMC diagnostics like $\hat{R}$).
- For posterior inference, MCMC is often preferable.

## Practical workflow: from hand-rolled MCMC to PPLs

The progression we have traced reflects a real scientific workflow:

1. **Prototype with hand-coded MCMC** (Chapter 11 onwards): Learn the model, understand the geometry of the posterior, diagnose what works and what doesn't.

2. **Apply parametrisation tricks**: Center and scale predictors, use whitening transforms, exploit posterior covariance structure.

3. **Upgrade to a gradient-based sampler**: Use HMC or NUTS to exploit smoothness of the log-posterior.

4. **Automate with Stan or JAGS**: Declare the model once, let the software handle sampling, diagnostics, and adaptation.

5. **Consider variational inference** if:
   - You need very fast approximate inference for exploration.
   - Your model is too large for MCMC.
   - You are happy accepting mean-field correlations.

## Failure modes and remedies

### Multimodal posteriors

If the target distribution has multiple distinct modes, MCMC may get stuck in one mode and miss others. JAGS and Stan both struggle with multimodality.

Remedies:

- Use parallel tempering (Chapter 10).
- Run many short chains from dispersed starting values.
- Reparametrise to reduce multimodality (e.g., via identifiability constraints).

### Discrete parameters

Stan requires all parameters to be continuous. If your model includes discrete latent variables (e.g., mixture component labels), you must marginalise them out analytically (integrate over all possible values) before passing the model to Stan.

JAGS handles discrete nodes naturally.

### Heavy-tailed posteriors

If the posterior has heavy tails (e.g., a mixture of normals with small weights), the leapfrog integrator in HMC can take very long steps and diverge. Divergent transitions are a warning.

Remedies:

- Increase `adapt_delta` (0.8 → 0.95 or higher).
- Increase `max_treedepth`.
- Reparametrise to reduce tail heaviness (e.g., use a Student-$t$ prior to allow more dispersed posteriors).

### Weak identifiability

If two parameters affect the likelihood in nearly the same way, the posterior is highly correlated or even degenerate. MCMC mixes slowly; variational inference underestimates uncertainty.

Remedy:

- Include informative priors that break the symmetry.
- Reparametrise (e.g., use a sum-of-effects rather than separate effects).

## What to report

For any Bayesian analysis using JAGS or Stan, report:

1. **Model specification**: Likelihood, priors, parametrisation.
2. **Sampling method**: Algorithm, number of chains, iterations, warmup.
3. **Convergence diagnostics**: Trace plots, $\hat{R}$, ESS, acceptance rate, divergent transitions (Stan only).
4. **Posterior summaries**: Mean or median, 95% credible intervals.
5. **Posterior predictions**: Regression curves with credible intervals; posterior predictive intervals if relevant.

Do not report precise posterior point estimates from a badly mixing chain—they are numerical artifacts, not inference.
