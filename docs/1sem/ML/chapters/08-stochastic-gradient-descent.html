9  Stochastic Gradient Descent

The goal of the learning process, as we know, is to identify the function that minimizes the risk function. In this chapter, we will introduce an algorithm capable of finding the minimum of a function iteratively and in a computationally efficient way, called Stochastic Gradient Descent.

9.1 Gradient Descent

The idea starts from the geometric meaning of the gradient of a convex function, (The convexity of the function is a hypothesis that is only briefly mentioned here but is essential for everything we will see. In the book, an entire chapter (the twelfth) is dedicated to the aspect of convexity.) which by definition “points” in the direction in which the function is increasing; this means that the opposite of this quantity will point in the direction of the minimum, which is precisely what we want.

Formally, we define the function \(f:\mathbb{R}^d\to\mathbb{R}\) and the vector \(\boldsymbol{w}\in\mathbb{R}^d\). The gradient is equal to

\[ \nabla f(\boldsymbol{w})=\left(\frac{\partial f(\boldsymbol{w})}{\partial w_1},\frac{\partial f(\boldsymbol{w})}{\partial w_2},\dots,\frac{\partial f(\boldsymbol{w})}{\partial w_d}\right) \]

By multiplying this gradient by a learning rate \(\eta\), we can control the size of the step that occurs at each iteration.

The algorithm, ultimately, is as follows:

Another motivation for choosing the gradient as a tool for finding the minimum is given by the Taylor series expansion of the function \(f(\boldsymbol{w})\). For a convex function, in fact, it is shown that

\[ f(\boldsymbol{w})\geqslant f\left(\boldsymbol{w_0}\right)+\left\langle\boldsymbol{w}-\boldsymbol{w_0},\nabla f\left(\boldsymbol{w_0}\right)\right\rangle \]

whatever the points \(\boldsymbol{w_0}\) and \(\boldsymbol{w}\) of the domain are.

Since the goal is to minimize \(f\left(\boldsymbol{w_0}\right)\), we want to get as close as possible to the case of equality:

\[ f(\boldsymbol{w})\simeq f\left(\boldsymbol{w_0}\right)+\left\langle\boldsymbol{w}-\boldsymbol{w_0},\nabla f\left(\boldsymbol{w_0}\right)\right\rangle \]

This approximation, as we know, is valid only when \(\boldsymbol{w}\) is close enough to \(\boldsymbol{w_0}\). At the same time, we want to find the absolute minimum of the function, so ultimately we will have to simultaneously minimize both the distance between \(\boldsymbol{w}\) and \(\boldsymbol{w_0}\) and the second member of (eq:stochastic-gradient-descent-gradient?) descent minorazione:

\[ \argmin_{\boldsymbol{w}}\left(\frac{1}{2}\norm{\boldsymbol{w}-\boldsymbol{w_0}}^2+\eta\left(f\left(\boldsymbol{w_0}\right)+\left\langle\boldsymbol{w}-\boldsymbol{w_0},\nabla f\left(\boldsymbol{w_0}\right)\right\rangle\right)\right) \]

where the factor \(\frac{1}{2}\) is introduced for mere calculation reasons and the factor \(\eta\) as a compromise parameter between the two terms.

We calculate this minimum by differentiating with respect to \(\boldsymbol{w}\) and setting the derivative to zero:

\[\begin{align*} &\frac{\partial}{\partial\boldsymbol{w}}\left(\frac{1}{2}\norm{\boldsymbol{w}-\boldsymbol{w_0}}^2+\eta\left(f\left(\boldsymbol{w_0}\right)+\left\langle\boldsymbol{w}-\boldsymbol{w_0},\nabla f\left(\boldsymbol{w_0}\right)\right\rangle\right)\right)=\\ &=\frac{1}{2}\frac{\partial}{\partial\boldsymbol{w}}\left[\left(\boldsymbol{w}-\boldsymbol{w_0}\right)^T\left(\boldsymbol{w}-\boldsymbol{w_0}\right)\right]+\eta\cancel{\frac{\partial}{\partial\boldsymbol{w}}f\left(\boldsymbol{w_0}\right)}+\eta\frac{\partial}{\partial\boldsymbol{w}}\left\langle\boldsymbol{w}-\boldsymbol{w_0},\nabla f\left(\boldsymbol{w_0}\right)\right\rangle=\\ &=\boldsymbol{w}-\boldsymbol{w_0}+\eta\nabla f\left(\boldsymbol{w_0}\right)=0 \end{align*}\]

From which we find:

\[ \boldsymbol{w}=\boldsymbol{w_0}-\eta\nabla f\left(\boldsymbol{w_0}\right) \]

which is exactly what we wrote in the Gradient Descent algorithm.

9.1.1 Convergence of the algorithm

Assuming \(\rho\)-Lipschitz continuity in addition to convexity, it is possible to relate the deviation of the algorithm’s output from the solution to the number of iterations of the algorithm itself.

Let \(f(\boldsymbol{w})\) be a convex and \(\rho\)-Lipschitz function. Let:

\[ \boldsymbol{w^*}=\argmin_{\boldsymbol{w}:\norm{\boldsymbol{w}}\leqslant B}f(\boldsymbol{w}) \]

If \(\eta=\sqrt{\frac{B^2}{\rho^2T}}\), then the Gradient Descent algorithm iterated \(T\) times will produce an output vector \(\boldsymbol{\bar{w}}\) such that

\[ f\left(\boldsymbol{\bar{w}}\right)-f\left(\boldsymbol{w^*}\right)\leqslant\frac{B\rho}{\sqrt{T}} \]

The proof is not required for the course. What is important for us is the immediate corollary to this theorem:

By iterating the Gradient Descent algorithm for \(T\geqslant\frac{B^2\rho^2}{\epsilon^2}\) with \(\epsilon>0\), then

\[ f\left(\boldsymbol{\bar{w}}\right)-f\left(\boldsymbol{w^*}\right)\leqslant\epsilon \]

We know that by iterating the algorithm \(T\) times, the difference between \(f\left(\boldsymbol{\bar{w}}\right)\) and \(f\left(\boldsymbol{w^*}\right)\) is bounded above by \(\frac{B\rho}{\sqrt{T}}\). For this to also be bounded by \(\epsilon\), it is necessary that

\[ \frac{B\rho}{\sqrt{T}}\leqslant\epsilon \]

Isolating \(T\):

\[ \sqrt{T}\geqslant\frac{B\rho}{\epsilon} \ \Longrightarrow \ T\geqslant\frac{B^2\rho^2}{\epsilon^2} \]

from which the thesis follows.

9.2 Stochastic variant

The exact calculation of the gradient requires the use of all the points in the training set for each iteration. This approach can be computationally expensive, so an alternative is often adopted that sacrifices some accuracy in favor of greater computational efficiency: the gradient is calculated using only one point for each iteration; this process converges to the minimum as long as the value chosen for the calculation of the gradient is drawn from a distribution that, on average, points in the direction of the gradient, so even if at each step the direction is not exactly directed towards the minimum, it will still be reached after a sufficiently high number of iterations.

The process just described is what gives the chapter its name: it is the Stochastic Gradient Descent (SGD), so called precisely because of its stochastic nature starting from the basis of Gradient Descent.

Let \(\boldsymbol{z}\) be the point drawn at the \(t\)-th iteration, what we want is that

\[ \mathbb{E}\left(\boldsymbol{z}|\boldsymbol{w}^{(t)}\right)=\nabla f\left(\boldsymbol{w}^{(t)}\right) \]

We will show later how it is possible to simply choose a random point from the learning set which is reshuffled at each iteration.

9.2.1 Comparison between the two versions of the algorithm

We have already explained the substantial difference: it is a saving of calculations on the one hand, and a lower overall accuracy on the other; as is easy to guess, this choice becomes all the more sensible the larger the training set becomes.

By choosing the stochastic version we would therefore have to deal with much less stable trajectories, as we have had the opportunity to observe in figure (fig:stochastic-gradient-descent-confronto-con-gd?). This is not a general problem in reality, since it also brings with it a positive aspect: that of escaping any local minima, which could lead the system to converge to a non-optimal solution.

In any case, we know some strategies that can help stabilize the trajectory in case it is excessively “noisy”:

  • Use of momentum, which we will discuss in more detail later.
  • Calculation of the output as the average of the points calculated at the various steps. It is possible to consider all the points obtained from the algorithm, but it is not recommended as the first vectors are usually very far from the minimum and would unbalance the final result too much; for this reason, the average is normally performed only on the last vectors, which are reasonably close to the optimal solution.
  • Choice of an adaptive learning rate, which increases and decreases the learning rate at each step based on a predetermined scheme that is a function of the slope of the gradient, the update frequency and other parameters.

9.2.2 Mini-batch variant

A further variant of Gradient Descent is the one that is configured as a middle ground between the two we have seen and which makes use of mini-batches, that is, sets of points chosen randomly from the training set.

Assuming the size of the mini-batch is \(k\), at each iteration of the algorithm the gradient is calculated through these \(k\) elements. It is clear, therefore, that the overall trajectory will be less stable than pure Gradient Descent but still less noisy than the stochastic variant. We thus find a compromise between accuracy and speed of calculation which is parameterized by \(k\), which is usually chosen by cross-validation. Precisely because of its role as a “middle ground”, this is the most used variant, especially in the training of very large neural networks.

9.3 Applications in optimization methods

9.3.1 SGD applied to ERM

What we are looking for is a way to obtain a random point \(\boldsymbol{z}\) such that the expected value conditioned by \(\boldsymbol{w}^{(t)}\) related to it is

\[ \mathbb{E}\left(\boldsymbol{z}|\boldsymbol{w}^{(t)}\right)=\nabla L_{\mathcal{D}}\left(\boldsymbol{w}^{(t)}\right) \]

being the risk function the quantity to be minimized.

We note that this equality is valid when we set the random vector equal to the gradient of the loss function calculated in \(\left(\boldsymbol{w^{(t)}},\boldsymbol{z}\right)\):

\[ \mathbb{E}\left(\boldsymbol{z}|\boldsymbol{w}^{(t)}\right)=\mathbb{E}_{\boldsymbol{z}\sim\mathcal{D}}\left(\nabla\ell\left(\boldsymbol{w}^{(t)},\boldsymbol{z}\right)\right)=\nabla \mathbb{E}_{\boldsymbol{z}\sim\mathcal{D}}\left(\ell\left(\boldsymbol{w}^{(t)},\boldsymbol{z}\right)\right)=\nabla L_{\mathcal{D}}\left(\boldsymbol{w}^{(t)}\right) \]

This quantity is also called estimator, or even unbiased estimator since the points are extracted directly from the set in our possession (training set) which is assumed to follow the same distribution \(\mathcal{D}\) of the data generation model.

There is also a theorem - related to the previous ones we have seen - which guarantees us an upper bound on the expected value of the loss function calculated in the output. We will not see the proof of this one either, as for the others.

Considering a convex and \(\rho\)-Lipschitz learning problem, iterating the Stochastic Gradient Descent method for \(T\geqslant\frac{B^2\rho^2}{\epsilon^2}\) times with a learning rate \(\eta=\sqrt{\frac{B^2}{\rho^2T}}\) we will obtain an output \(\boldsymbol{\bar{w}}\) such as to satisfy

\[ \mathbb{E}\left(L_{\mathcal{D}}\left(\boldsymbol{\bar{w}}\right)\right)\leqslant\min_{\boldsymbol{w}\in\mathcal{H}}L_{\mathcal{D}}(\boldsymbol{w})+\epsilon \hspace{0.65cm} \forall\epsilon \]

where \(B\) is such that \(\norm{\boldsymbol{w}}\leqslant B \ \forall\boldsymbol{w}\).

In essence, we can therefore state that SGD is - on average - an agnostic PAC learner, according to what we know from section (sec:quando-lerm-e-un-agnostic-pac-learner?).

9.3.2 SGD applied to RLM

Applying the same reasoning as before to the regularized loss function, the regularization term will be added, whose gradient is

\[ \nabla\left(\frac{\lambda}{2}\norm{\boldsymbol{w}}^2\right)=\lambda\boldsymbol{w} \]

At the \(t\)-th iteration the output will be:

\[ \boldsymbol{w^{(t+1)}}\to\boldsymbol{w^{(t)}}-\eta\left(\boldsymbol{z^{(t)}}+\lambda\boldsymbol{w^{(t)}}\right) \]

It can be shown that this iteration can also be written as

\[ \boxed{\boldsymbol{w^{(t+1)}}\to-\eta\sum_{i=1}^t\boldsymbol{z^{(i)}}} \]

Another thing that can be shown is that the regularized loss function is a \(\lambda\)-strongly convex function, (As anticipated, although convexity plays a key role in this whole discussion, it has not been introduced and explored properly, so this information is simply given without further explanation. We can say that, intuitively, it is a function that grows at least like a quadratic function and whose growth is parameterized by \(\lambda\).), and for a function of this type it is convenient to choose a learning rate inversely proportional to the step of the iteration \(t\); to be precise:

\[ \eta=\frac{1}{\lambda t} \ \Longrightarrow \ \boldsymbol{w^{(t+1)}}\to-\frac{1}{\lambda t}\sum_{i=1}^t\boldsymbol{z^{(i)}} \]

Finally, if the loss \(f\) is \(\rho\)-Lipschitz, the difference between the expected value relative to the final output and the true error is bounded by a known quantity:

\[ \mathbb{E}\left(f\left(\boldsymbol{\bar{w}}\right)\right)-f\left(\boldsymbol{w^*}\right)\leqslant\frac{4\rho^2}{\lambda T}(1+\log T) \]

9.3.3 SGD applied to soft SVM

We know that the loss function for soft SVM is the hinge loss:

\[ \ell^{hinge}((\boldsymbol{w},b),(\boldsymbol{x},y))=\max(0,1-y(\sprod{x}{w}+b)) \]

This function, in principle, is not differentiable; nevertheless, we can still calculate the sub-gradients:

\[ \nabla\ell^{hinge}((\boldsymbol{w},b),(\boldsymbol{x},y))= \begin{cases} 0 \ \text{if} \ 1-y(\sprod{x}{w}+b)\leqslant0\\ -y\boldsymbol{x} \ \text{if} \ 1-y(\sprod{x}{w}+b)>0 \end{cases} \]

Based on this, we can write the algorithm that produces the final output.

9.4 Momentum

The main problem related to the Stochastic Gradient Descent process is that of converging to local minima that do not represent the optimal solutions for the problem. One way to regulate this is to play with the learning rate parameter, increasing or decreasing it based on how large we want the “jumps” performed by the algorithm to be; we have also mentioned that these “jumps” do not have to be constant but can vary during the iterations, and how it is often convenient that \(\eta\propto\frac{1}{t}\).

There is another method to facilitate the convergence of the algorithm to the global minimum, which is the use of momentum. We can think of the process, in fact, in analogy to a ball rolling on a surface full of curves: in the vicinity of a local minimum, if the ball has a sufficiently high impulse given by inertia, it will be able to overcome this valley and proceed further. In a completely analogous way, we can think of considering not only the gradient of the function, which indicates the privileged direction of optimization, but also the inertia itself given by the direction indicated at the previous step. We will therefore define a parameter \(\gamma\) such that

\[ \boldsymbol{z}^{(t)}=\gamma\boldsymbol{z}^{(t-1)}+(1-\gamma)\nabla L_{\mathcal{D}}\left(\boldsymbol{w}^{(t)}\right) \]

When \(\gamma=0\) the algorithm is a pure SGD, as the weight of the inertia is zero. The closer \(\gamma\) gets to \(1\), however, the more the inertia will count at the expense of the direction indicated by the gradient.

\begin{algorithm}[H]
    \caption*{**Gradient Descent** (pseudocode)}
    \begin{algorithmic}[1]
        \State $\boldsymbol{w} \gets (0,\dots,0)$
        \State
        \For{$t=1,2,\dots,T-1$}
            \State $\boldsymbol{w}\gets\boldsymbol{w}-\eta\nabla f(\boldsymbol{w})$
        \EndFor
        \State
        \State \Return $\boldsymbol{w}$
    \end{algorithmic}
 \end{algorithm}

 $$
    f(\boldsymbol{w})\geqslant f\left(\boldsymbol{w_0}\right)+\left\langle\boldsymbol{w}-\boldsymbol{w_0},\nabla f\left(\boldsymbol{w_0}\right)\right\rangle

\begin{align*}

\end{align*}

From which we find:

 $$
    \boldsymbol{w}=\boldsymbol{w_0}-\eta\nabla f\left(\boldsymbol{w_0}\right)

### Convergence of the algorithm

Assuming $\rho$-Lipschitz continuity in addition to convexity, it is possible to relate the deviation of the algorithm's output from the solution to the number of iterations of the algorithm itself.

::: {.theorem name="Theorem on the convergence of Gradient Descent"}
Let $f(\boldsymbol{w})$ be a convex and $\rho$-Lipschitz function. Let:

$$
    \boldsymbol{w^*}=\argmin_{\boldsymbol{w}:\norm{\boldsymbol{w}}\leqslant B}f(\boldsymbol{w})
$$

$$
    f\left(\boldsymbol{\bar{w}}\right)-f\left(\boldsymbol{w^*}\right)\leqslant\frac{B\rho}{\sqrt{T}}
$$
:::

The proof is not required for the course. What is important for us is the immediate corollary to this theorem:

::: {.theorem name="Corollary"}
By iterating the Gradient Descent algorithm for $T\geqslant\frac{B^2\rho^2}{\epsilon^2}$ with $\epsilon>0$, then

$$
    f\left(\boldsymbol{\bar{w}}\right)-f\left(\boldsymbol{w^*}\right)\leqslant\epsilon
$$

\exrule

We know that by iterating the algorithm $T$ times, the difference between $f\left(\boldsymbol{\bar{w}}\right)$ and $f\left(\boldsymbol{w^*}\right)$ is bounded above by $\frac{B\rho}{\sqrt{T}}$. For this to also be bounded by $\epsilon$, it is necessary that

$$
    \frac{B\rho}{\sqrt{T}}\leqslant\epsilon
$$

Isolating $T$:

$$
    \sqrt{T}\geqslant\frac{B\rho}{\epsilon} \ \Longrightarrow \ T\geqslant\frac{B^2\rho^2}{\epsilon^2}

Finally, if the loss $f$ is $\rho$-Lipschitz, the difference between the expected value relative to the final output and the true error is bounded by a known quantity:

 $$
    \mathbb{E}\left(f\left(\boldsymbol{\bar{w}}\right)\right)-f\left(\boldsymbol{w^*}\right)\leqslant\frac{4\rho^2}{\lambda T}(1+\log T)
 $$

### SGD applied to soft SVM

We know that the loss function for soft SVM is the hinge loss:

 $$
    \ell^{hinge}((\boldsymbol{w},b),(\boldsymbol{x},y))=\max(0,1-y(\sprod{x}{w}+b))
 $$

This function, in principle, is not differentiable; nevertheless, we can still calculate the **sub-gradients**:

 $$
    \nabla\ell^{hinge}((\boldsymbol{w},b),(\boldsymbol{x},y))=
    \begin{cases}
        0 \ \text{if} \ 1-y(\sprod{x}{w}+b)\leqslant0\\
        -y\boldsymbol{x} \ \text{if} \ 1-y(\sprod{x}{w}+b)>0
    \end{cases}
 $$

Based on this, we can write the algorithm that produces the final output.

\begin{algorithm}[H]
    \caption*{**Stochastic Gradients Descent applied to soft SVM** (pseudocode)}
    \begin{algorithmic}[1]
        \State $\boldsymbol{z}\gets(0,0,\dots,0)$
        \State
        \For{$t=1$ to $T$}
            \State
            \State $\boldsymbol{w},b\gets\boldsymbol{z} / (\lambda*t)$
            \State
            \State $i\gets UniformExtraction(1,m)$\Comment{Random extraction from the training set}
            \State $\boldsymbol{x},y\gets FromTrainingSet(i)$
            \State $class\gets1-y*(ScalarProduct( oldsymbol{x}, oldsymbol{w})+b)$
            \State
            \If{$class>0$}
                \State $\boldsymbol{z}\gets\boldsymbol{z}+y*\boldsymbol{x}$
            \EndIf
            \State
        \EndFor
        \State
        \State
        \Return{$\boldsymbol{w},b$}
    \end{algorithmic}
\end{algorithm}

## Momentum

The main problem related to the Stochastic Gradient Descent process is that of converging to local minima that do not represent the optimal solutions for the problem. One way to regulate this is to play with the learning rate parameter, increasing or decreasing it based on how large we want the "jumps" performed by the algorithm to be; we have also mentioned that these "jumps" do not have to be constant but can vary during the iterations, and how it is often convenient that $\eta\propto\frac{1}{t}$.

There is another method to facilitate the convergence of the algorithm to the global minimum, which is the use of **momentum**. We can think of the process, in fact, in analogy to a ball rolling on a surface full of curves: in the vicinity of a local minimum, if the ball has a sufficiently high impulse given by inertia, it will be able to overcome this valley and proceed further. In a completely analogous way, we can think of considering not only the gradient of the function, which indicates the privileged direction of optimization, but also the inertia itself given by the direction indicated at the previous step. We will therefore define a parameter $\gamma$ such that

 $$
    \boldsymbol{z}^{(t)}=\gamma\boldsymbol{z}^{(t-1)}+(1-\gamma)\nabla L_{\mathcal{D}}\left(\boldsymbol{w}^{(t)}\right)
 $$

When $\gamma=0$ the algorithm is a pure SGD, as the weight of the inertia is zero. The closer $\gamma$ gets to $1$, however, the more the inertia will count at the expense of the direction indicated by the gradient.
