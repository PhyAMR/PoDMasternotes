16  Reinforcement Learning

16.1 Motivation: The Agent-Environment Loop

Unlike supervised learning (labels provided) and unsupervised learning (structure discovery), reinforcement learning (RL) uses rewards as the training signal. An agent interacts with an environment, takes actions, receives rewards, and transitions to new states. The goal is to learn a policy (decision rule) that maximizes cumulative reward over time.

Key challenges unique to RL:

  • Credit assignment: How do we attribute delayed rewards to earlier actions? (The reward for winning a chess game comes long after the opening move.)
  • Exploration vs. exploitation: Should we exploit what we know works, or explore to discover better strategies?
  • Non-stationarity: The distribution of states and rewards may change as we learn.

16.2 Markov Decision Processes

The foundation of RL is the Markov Decision Process (MDP), which formalizes sequential decision-making under uncertainty.

An MDP is a tuple \((S, A, P, R, \gamma)\):

  • State space \(S\): Set of possible states the environment can occupy.
  • Action space \(A\): Set of actions the agent can take.
  • Transition dynamics \(P\): For each state \(s\) and action \(a\), \(P(s' | s, a)\) is the probability of transitioning to state \(s'\).
  • Reward function \(R\): \(R(s, a, s')\) or \(R(s, a)\) is the immediate reward for taking action \(a\) in state \(s\).
  • Discount factor \(\gamma \in [0, 1)\): Weighting future vs. immediate rewards.

The Markov property states: \(P(s_{t+1} | s_t, a_t, s_{t-1}, a_{t-1}, \ldots) = P(s_{t+1} | s_t, a_t)\). The future depends only on the current state and action, not history.

Intuition: The MDP captures the essential structure of decision-making: state (where we are), action (what we do), consequence (next state), and payoff (reward).

16.3 Return and Discount Factor

When an agent follows a sequence of actions over time, it accumulates rewards. We define the return (cumulative discounted reward) starting from time \(t\):

\[G_t = r_{t+1} + \gamma r_{t+2} + \gamma^2 r_{t+3} + \cdots = \sum_{k=0}^{\infty} \gamma^k r_{t+k+1}.\]

The discount factor \(\gamma\) has two interpretations:

  • Time preference: \(\gamma\) close to 0 prioritizes immediate rewards; \(\gamma\) close to 1 values long-term payoffs equally.
  • Convergence: For infinite-horizon problems, \(\gamma < 1\) ensures the sum converges (assuming bounded rewards).

For finite-horizon problems (where the episode ends after \(T\) steps), we can use \(\gamma = 1\).

16.4 Value Functions

A policy \(\pi\) maps states to actions (deterministically) or to probability distributions over actions (stochastically).

  • State-value function under policy \(\pi\): \[V^\pi(s) = \mathbb{E}_\pi[G_t | s_t = s] = \mathbb{E}_\pi\left[\sum_{k=0}^{\infty} \gamma^k r_{t+k+1} \,\Big|\, s_t = s\right].\] The expected return starting from state \(s\) and following \(\pi\).

  • Action-value function (Q-value) under policy \(\pi\): \[Q^\pi(s, a) = \mathbb{E}_\pi[G_t | s_t = s, a_t = a].\] The expected return after taking action \(a\) in state \(s\), then following \(\pi\).

The optimal value functions are: \[V^*(s) = \max_\pi V^\pi(s), \quad Q^*(s, a) = \max_\pi Q^\pi(s, a).\]

Intuition: Value functions quantify the goodness of states and state-action pairs. An agent aims to reach high-value states and choose high-value actions.

16.5 Bellman Equation

The recursive structure of returns yields the Bellman equation, which relates a state’s value to its successors’ values:

For any policy \(\pi\) and any state \(s\):

\[V^\pi(s) = \sum_a \pi(a|s) \sum_{s'} P(s'|s,a) \left[R(s,a,s') + \gamma V^\pi(s')\right].\]

For the optimal policy:

\[V^*(s) = \max_a \left[R(s,a) + \gamma \sum_{s'} P(s'|s,a) V^*(s')\right].\]

The second form is the Bellman optimality equation: the optimal value of a state equals the immediate reward of the best action plus the (discounted) optimal value of the resulting state.

Intuition: Value is a sum of immediate payoff and future payoff. This decomposition enables dynamic programming—we can solve for value functions without full lookahead.

16.6 Dynamic Programming Solutions

When the MDP is fully known (we have \(P, R\)), we can solve for the optimal policy using dynamic programming.

16.6.1 Policy Evaluation

Given a policy \(\pi\), compute its value function by iterating the Bellman equation:

\[V_{k+1}(s) = \sum_a \pi(a|s) \sum_{s'} P(s'|s,a) \left[R(s,a,s') + \gamma V_k(s')\right].\]

Repeat until convergence: \(V_k \approx V^\pi\).

16.6.2 Policy Improvement

Given a value function, extract a greedy policy:

\[\pi'(s) = \arg\max_a \left[R(s,a) + \gamma \sum_{s'} P(s'|s,a) V(s')\right].\]

16.6.3 Policy Iteration

Alternate between evaluation and improvement until the policy stabilizes:

  1. Initialize \(\pi\) arbitrarily.
  2. Repeat:
    • Policy Evaluation: Compute \(V^\pi\) (or approximate it).
    • Policy Improvement: \(\pi \gets \arg\max_a [\cdots]\) using \(V^\pi\).
    • Stop if \(\pi\) doesn’t change.

Complexity: Polynomial in state and action space sizes. Convergence: Guaranteed to the optimal policy in finite MDPs.

16.6.4 Value Iteration

Alternatively, directly iterate the Bellman optimality equation:

\[V_{k+1}(s) = \max_a \left[R(s,a) + \gamma \sum_{s'} P(s'|s,a) V_k(s')\right].\]

Repeat until convergence. This is simpler (single loop) and also converges to \(V^*\).

16.7 Model-Free Temporal Difference Learning

In practice, the MDP is unknown—we don’t have \(P\) or \(R\). Instead, we learn from experience. Temporal Difference (TD) methods combine ideas from Monte Carlo methods and dynamic programming.

16.7.1 TD(0) Update

When the agent observes a transition \((s, a, r, s')\) with immediate reward \(r\) and next state \(s'\), update:

\[V(s) \leftarrow V(s) + \alpha \left[r + \gamma V(s') - V(s)\right],\]

where \(\alpha\) is the learning rate (typically \(\alpha \in (0, 1)\) and decaying over time).

The term \(\delta = r + \gamma V(s') - V(s)\) is the temporal difference error—the mismatch between predicted and actual return. We nudge \(V(s)\) toward \(r + \gamma V(s')\), the one-step-ahead estimate.

Convergence: Under mild conditions on learning rates and state/action visitation, TD(0) converges to the true value function \(V^\pi\) with probability 1.

16.7.2 SARSA: On-Policy TD Control

To learn the optimal policy, we estimate Q-values on-policy. SARSA (State-Action-Reward-State-Action) learns \(Q(s, a)\) by updating after each experience:

\[Q(s, a) \leftarrow Q(s, a) + \alpha \left[r + \gamma Q(s', a') - Q(s, a)\right],\]

where \((s', a')\) is the next state-action pair chosen by the current policy (typically \(\epsilon\)-greedy).

Limitation: Because SARSA uses the action chosen by the current policy (which may be exploratory), it is on-policy. It tends to learn a policy that avoids risky paths, even if they lead to high rewards. (SARSA avoids the “optimal move that risks a penalty.”)

16.7.3 Q-Learning: Off-Policy TD Control

Q-learning decouples the action used for exploration from the action used for updating:

\[Q(s, a) \leftarrow Q(s, a) + \alpha \left[r + \gamma \max_{a'} Q(s', a') - Q(s, a)\right].\]

Here, the update uses the greedy best action \(\arg\max_{a'} Q(s', a')\) regardless of which action we actually took. This is off-policy: we can learn the optimal policy from data collected under a different (e.g., more exploratory) policy.

Convergence: Under standard assumptions, Q-learning converges to \(Q^*\) with probability 1.

Advantage: Q-learning often outperforms SARSA in practice because it learns the optimal policy even when following an exploratory policy.

16.8 Exploration vs. Exploitation

A fundamental trade-off in RL:

  • Exploitation: Choose the action with the highest estimated Q-value (greedy).
  • Exploration: Try other actions to learn if they’re better.

16.8.1 \(\epsilon\)-Greedy Exploration

A simple strategy: - With probability \(1 - \epsilon\), choose \(a = \arg\max_{a'} Q(s, a')\) (exploit). - With probability \(\epsilon\), choose a random action (explore).

Typically, \(\epsilon\) decays over time (e.g., \(\epsilon_t = \epsilon_0 / t\)), prioritizing exploration early and exploitation later.

16.8.2 Softmax Exploration

Choose action \(a\) with probability:

\[P(a | s) = \frac{\exp(Q(s, a) / \tau)}{\sum_{a'} \exp(Q(s, a') / \tau)},\]

where \(\tau\) is a temperature parameter. High \(\tau\) makes probabilities uniform (more exploration); low \(\tau\) concentrates on high-Q actions (more exploitation).

16.9 Deep Q-Networks (DQN) and Curse of Dimensionality

16.9.1 The Curse of Dimensionality

As state space grows exponentially with dimensionality, tabular methods (storing \(Q(s, a)\) for each state-action pair) become infeasible. For a \(20 \times 20\) pixel image with 256 grayscale levels, there are \(256^{400}\) possible states—astronomically large.

16.9.2 Function Approximation and Deep Learning

Instead of a table, use a neural network to approximate \(Q(s, a) \approx Q_\theta(s, a)\), where \(\theta\) are network weights. Update \(\theta\) using gradient descent on the TD error:

\[\theta \leftarrow \theta - \alpha \nabla_\theta \left[r + \gamma \max_{a'} Q_{\theta'}(s', a') - Q_\theta(s, a)\right]^2.\]

16.9.3 DQN Innovations

Combining neural networks with Q-learning naively causes instability (samples are correlated and non-stationary). Deep Q-Networks (DQN) address this with:

Remark.

  • Experience replay: Store transitions \((s, a, r, s')\) in a buffer. Sample minibatches uniformly at random to break temporal correlations, making training more stable.
  • Target network: Maintain a separate “target” network \(Q_{\theta^-}\) with frozen weights, updated infrequently (every \(C\) steps) to \(Q_\theta\). Use \(Q_{\theta^-}\) in the Bellman target \(r + \gamma \max_{a'} Q_{\theta^-}(s', a')\) to reduce non-stationarity.

These techniques allowed DQN to learn directly from raw pixels on Atari games, achieving superhuman performance on many games.

16.10 Worked Example: Gridworld Value Iteration

Consider a \(3 \times 3\) gridworld where the agent starts in the center and aims to reach the top-left corner (reward \(+1\)) or bottom-right corner (reward \(-1\)). All other transitions have reward \(0\). The agent can move up, down, left, or right; moves hitting walls stay in place.

Using value iteration with \(\gamma = 0.9\):

Iteration 0: \(V(s) = 0\) for all \(s\).

Iteration 1: - Top-left corner: \(V(s) = \max_a [1 + 0.9 \cdot 0] = 1\). - Bottom-right corner: \(V(s) = \max_a [-1 + 0.9 \cdot 0] = -1\). - Others: \(V(s) = 0\) (no immediate reward; neighbors have value 0).

Iteration 2: - Neighbors of corners now have value \(0 + 0.9 \times (\text{neighbor value}) > 0\) or \(< 0\). - Values propagate outward.

After convergence, the optimal policy tells the agent: move toward the top-left corner (positive reward) and away from the bottom-right (negative). The center cell has intermediate value reflecting the balance of both corners.

16.11 Policy Gradient Methods (Brief Mention)

Rather than learning values, policy gradient methods directly optimize the policy \(\pi_\theta(a | s)\) by gradient ascent on the expected return:

\[\theta \leftarrow \theta + \alpha \nabla_\theta \mathbb{E}[G_t | \pi_\theta].\]

REINFORCE is a classic on-policy method. Advantages: Can handle continuous action spaces, stochastic policies. Disadvantages: Higher variance, slower convergence than value-based methods. Modern algorithms like actor-critic and trust region methods (PPO, TRPO) combine policy gradient with value estimates to reduce variance.

16.12 Summary

Reinforcement learning addresses sequential decision-making under uncertainty:

  • MDPs formalize the problem with states, actions, transitions, and rewards.
  • Value functions quantify expected long-term payoff; the Bellman equation decomposes them recursively.
  • Dynamic programming solves known MDPs via policy iteration or value iteration.
  • Temporal difference learning (TD, SARSA, Q-learning) learns from experience without knowing the MDP.
  • Exploration vs. exploitation is fundamental; \(\epsilon\)-greedy and softmax offer practical trade-offs.
  • Deep Q-Networks combine function approximation with RL, enabling learning on high-dimensional problems (e.g., raw images).
  • Policy gradient methods directly optimize the policy, offering flexibility for continuous control.

RL has achieved remarkable results in games (AlphaGo, Atari), robotics, and autonomous systems, but remains challenging due to sample efficiency, stability, and the exploration-exploitation trade-off.

When the MDP is fully known (we have $P, R$), we can solve for the optimal policy using **dynamic programming**.

Given a policy $\pi$, compute its value function by iterating the Bellman equation:

$$V_{k+1}(s) = \sum_a \pi(a|s) \sum_{s'} P(s'|s,a) \left[R(s,a,s') + \gamma V_k(s')\right].$$

Given a value function, extract a greedy policy:

Alternate between evaluation and improvement until the policy stabilizes:

1. **Initialize** $\pi$ arbitrarily.
2. **Repeat**:
   - Policy Evaluation: Compute $V^\pi$ (or approximate it).
   - Policy Improvement: $\pi \gets \arg\max_a [\cdots]$ using $V^\pi$.
