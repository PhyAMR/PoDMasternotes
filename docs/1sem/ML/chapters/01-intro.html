2  Introduction & ML Models

2.1 What is machine learning?

Machine learning is a set of methods, tools, and algorithms that gives a computer system the ability to “learn” certain behaviors from a set of data without being explicitly programmed to do so.

In classical programming, on the contrary, it is necessary for the programmer to specify in advance what the machine must execute: to classify certain objects, for example, he can write that if a given image has less than \(30\%\) of red pixels then it belongs to a certain class \(A\), if this number is between \(30\%\) and \(50\%\) then it belongs to class \(B\) and so on.

In short:

In machine learning, the programmer’s task is to create algorithms and models through which a computer system is trained starting from a set of data that is provided to it.

2.2 Types of Learning

We divide the possible types of learning into two families:

  • Supervised learning: learning takes place through a data set to which appropriate labels are assigned, so to each input provided to the system there is also an associated label that allows the categorization of the input itself. Starting from all the inputs divided by categories, the system must then be able to generalize so that inputs of the same type are autonomously recognized and divided.
  • Unsupervised learning: the system learns by identifying patterns in the data that do not have labels of any kind.

The example given by the professor in class is useful for understanding the difference between the two families in practice; let’s imagine providing images of dogs and cats as a data set:

\item[**\textbullet**] In the case of supervised learning, each image will be assigned the label *dog* or the label *cat*, so that the system is able to understand what distinguishes the two categories and is then able to state whether a new image provided without a label is that of a dog or a cat.
\item[**\textbullet**] In the case of unsupervised learning, the images will not be labeled and it will be up to the system to find categories in which to insert the images by identifying appropriate patterns; in this case it could distinguish the images of dogs from those of cats, but it could also distinguish the images of chihuahuas from those of great danes, treating them as different categories.

For the entire first part of the course, we will only refer to supervised learning.

2.3 Data Representation

A machine learning algorithm has access to three sets:

  • Domain \(\mathcal{X}\): this is the set of all possible inputs

    An element belonging to the domain (\(x\in\mathcal{X}\)) is called a domain point (In English, besides domain point, it can also be called instance.) and is typically represented as a point in a multidimensional space, that is, as a vector of features (This is a vector that orders all the numerical characteristics of the element; in English it is called vector of features.)

    Example: If the input is an image with a resolution of \(100\times100\), then the domain consists of all the possible pixels that can form an image. The set is therefore composed of all the possible positions in the grid (i.e., ), all three possible channels (red, blue, and green), and the various shades of color (256 in the case of 8-bit pixels).

  • Set of labels \(\mathcal{Y}\): this is the set of all possible outputs

    The simplest case (which will be the one considered for most of the course) is that of binary classification in which the possible outputs are only two, and therefore we have \(\mathcal{Y}=\left\{0,1\right\}\).

  • Learning set \(\mathcal{S}\): this is a finite sequence of labeled domain points, and therefore a subset of \(\mathcal{X}\times\mathcal{Y}\)

    This is the input of the machine learning algorithm, since through this set of pairs \(\left(\left(x_1,y_1\right),\dots,\left(x_m,y_m\right)\right)\) it will be able to identify a law that follows what is indicated by these pairs.

2.3.1 Predictions with respect to data

Further characteristics associated with the algorithm are the following:

  • Prediction rule \(h:\mathcal{X}\to\mathcal{Y}\): the function that maps the elements of \(\mathcal{X}\) to \(\mathcal{Y}\), thus predicting the labels of a generic domain of points

    This is none other than the output of the algorithm, also called predictor, hypothesis, or classifier. The prediction rule specifically associated with the learning set \(\mathcal{S}\) is generally indicated by \(A(\mathcal{S})\).

  • Data generation model: we assume that the domain points \(x_i\) present in the learning set are generated from a probability distribution \(\mathcal{D}\) on \(\mathcal{X}\) and that the labels \(y_i\) associated with the points are realized from a function \(f\) such that \(y_i=f\left(x_i\right)\)

    It is necessary to underline that the algorithm does not know the distribution \(\mathcal{D}\) nor, much less, the labeling function \(f\); the latter, in fact, is precisely what the algorithm aims to identify.

  • Measure of success: how much one can minimize the error of \(h\) with respect to the “actual” prediction function \(f\), that is, the probability that, taking any \(x_i\) from the distribution \(\mathcal{D}\), it occurs that \(h\left(x_i\right)\neq f\left(x_i\right)\)

    Formally, given a subset \(A\subset \mathcal{X}\), the probability distribution \(\mathcal{D}\) on \(A\), which we can denote by \({\mathcal{D}}(A)\), represents the probability that a point \(x\) belongs precisely to \(A\); equivalently, one can think of \(A\) as an event and \(\pi(x)\) as a function that is equal to \(1\) when \(x\in A\), that is, when the event occurs, and equal to \(0\) otherwise: in this case we can write \({\mathcal{D}}(A)\) as \(\mathbb{P}_{x\sim\mathcal{D}}[\pi(x)]\).

    Having made these premises, we can define the loss function, also called true error or true risk, as

    \[ \boxed{L_{\mathcal{D},f}(h)={\mathcal{D}}(x:h(x)\neq f(x))=\mathbb{P}_{x\sim\mathcal{D}}[h(x)\neq f(x)]} \]

2.4 Empirical Risk Minimization

The purpose of the algorithm, as mentioned, is to minimize the error, and therefore the loss function. The problem is that this depends on the distribution \(\mathcal{D}\) and the function \(f\) which are not known to the algorithm; the question therefore becomes: how is it possible to estimate this error?

The most logical thing to do is to rely on the learning set \(S\), since in that case it is possible to compare with known results, as the labels are provided as input together with the points. We therefore define the training error as follows:

\[ L_S(h)=\frac{| i:h\left(x_i\right)\neq y_i,1\leqslant i\leqslant m|}{m} \]

that is, as the ratio between the number of incorrect predictions and the number of elements in \(S\), also called the training sample.

This learning paradigm, which consists in finding a predictor \(h\) that minimizes \(L_S(h)\), is called empirical risk minimization or ERM (**Empirical ).

2.4.1 Finite Class of Hypotheses

The most natural question that follows all this reasoning is:

To understand this, let’s study a simple but effective example: let’s consider \(\mathcal{X}\) as a square within which we can find a certain number of points \(x\) uniformly distributed over the entire square; the function that labels the points is very simple:

\[ f(x)= \begin{cases} 0 \ \ \ \ \text{if x is on the upper part}\\ 1 \ \ \ \ \text{if x is on the lower part} \end{cases} \]

Since the distribution is uniform over the entire square, this means that the points have the same probability of falling on one side or the other and, consequently, of being labeled with \(0\) or with \(1\).

Let’s imagine we have the following learning set:

A possible predictor obtained from this \(S\) could distinguish between the points on the left side and the points on the right side:

\[ h_{\mathcal{S}}(x)= \begin{cases} 0 \ \ \ \ \text{if x is on the left part}\\ 1 \ \ \ \ \text{if x is on the right part} \end{cases} \]

The training error \(L_{\mathcal{S}}(h)\) is clearly zero, since all the points of \(\mathcal{S}\) are correctly labeled through the prediction rule \(h_{\mathcal{S}}\). The true error, however, is equal to

\[ L_{\mathcal{D},f}\left(h_{\mathcal{S}}\right)=\frac{1}{2} \]

since a point extracted in the lower left or upper right part (half of the domain) would be erroneously labeled. This is a very high error, being exactly equal to the one that would be obtained by randomly choosing the label to associate with the point: we can deduce that, in general, empirical risk minimization alone is not a good learning method for calculating the risk.

When an event of this type occurs, that is, an excellent predictor relative to the learning set but not with respect to a generic data set, it means that we are in the presence of overfitting.

So at this point we ask ourselves:

The first thing that could be done is to apply ERM to a restricted class of possible hypotheses, because whatever the learning set \(\mathcal{S}\) is, it is always possible to overfit: in the example just seen, in fact, we could have chosen curved regions, therefore not necessarily delimited by straight lines; choosing straight lines, in this regard, could be considered a restriction of the set of hypotheses.

From a rigorous point of view, we make the following assumptions:

  • We consider the finite class of hypotheses \(\mathcal{H}\).

    Informally, we can write \(\mathcal{H}<\infty\), in the sense that there exists a natural number \(M\) such that the elements belonging to \(\mathcal{H}\), that is, the possible hypotheses, are less than \(M\) in number.

  • We assume realizability is valid, that is, the property that tells us that

    \[ \exists h^*\in\mathcal{H} : L_{\mathcal{D},f}\left(h^*\right)=0 \]

    It is important to note that this condition also implies that \(L_{\mathcal{S}}\left(h^*\right)=0\), because if it is valid for any set distributed according to \(\mathcal{D}\), it will be even more so for the learning set \(\mathcal{S}\) from which the algorithm learns.

  • The points of the set \(\mathcal{S}\) must be i.i.d., that is, independent and identically distributed according to the distribution \(\mathcal{D}\).

    This last hypothesis is very important, because if the points of the learning set are correlated in some way with each other, this results in an obstacle to the search for an algorithm that finds the right pattern in the data. We indicate this statement with \(\mathcal{S}\sim\mathcal{D}^m\), where \(m\) is the number of points extracted from \(\mathcal{S}\).

## Empirical Risk Minimization

The purpose of the algorithm, as mentioned, is to minimize the error, and therefore the loss function. The problem is that this depends on the distribution $\mathcal{D}$ and the function $f$ which are not known to the algorithm; the question therefore becomes: how is it possible to estimate this error?

The most logical thing to do is to rely on the learning set $S$, since in that case it is possible to compare with known results, as the labels are provided as input together with the points. We therefore define the **training error** as follows:

$$
    L_S(h)=\frac{| i:h\left(x_i\right)\neq y_i,1\leqslant i\leqslant m|}{m}
$$

that is, as the ratio between the number of incorrect predictions and the number of elements in $S$, also called the *training sample*.

This learning paradigm, which consists in finding a predictor $h$ that minimizes $L_S(h)$, is called **empirical risk minimization** or **ERM** (***E*mpirical \textbf{R**isk **M**inimization}).

The most natural question that follows all this reasoning is:

\begin{center}
    *is the training error a good estimate of the true error?*
\end{center}

To understand this, let's study a simple but effective example: let's consider $\mathcal{X}$ as a square within which we can find a certain number of points $x$ uniformly distributed over the entire square; the function that labels the points is very simple:

$$
    f(x)=
    \begin{cases}
        0 \ \ \ \ \text{if x is on the upper part}\\
        1 \ \ \ \ \text{if x is on the lower part}
    \end{cases}
$$

Since the distribution is uniform over the entire square, this means that the points have the same probability of falling on one side or the other and, consequently, of being labeled with $0$ or with $1$.

Let's imagine we have the following learning set:

A possible predictor obtained from this $S$ could distinguish between the points on the left side and the points on the right side:

$$
    h_{\mathcal{S}}(x)=
    \begin{cases}
        0 \ \ \ \ \text{if x is on the left part}\\
        1 \ \ \ \ \text{if x is on the right part}
    \end{cases}
$$

The training error $L_{\mathcal{S}}(h)$ is clearly zero, since all the points of $\mathcal{S}$ are correctly labeled through the prediction rule $h_{\mathcal{S}}$. The true error, however, is equal to

$$
    L_{\mathcal{D},f}\left(h_{\mathcal{S}}\right)=\frac{1}{2}
$$

since a point extracted in the lower left or upper right part (half of the domain) would be erroneously labeled. When an event of this type occurs, that is, an excellent predictor relative to the learning set but not with respect to a generic data set, it means that we are in the presence of **overfitting**.

So at this point we ask ourselves:

\begin{center}
    ***when* can ERM be generalized to any set?**
\end{center}

The first thing that could be done is to apply ERM to a **restricted class** of possible hypotheses, because whatever the learning set $\mathcal{S}$ is, it is always possible to overfit: in the example just seen, in fact, we could have chosen curved regions, therefore not necessarily delimited by straight lines; choosing straight lines, in this regard, could be considered a restriction of the set of hypotheses.

- We consider the **finite** class of hypotheses $\mathcal{H}$.

- We assume **realizability** is valid, that is, the property that tells us that
    
    $$
        \exists h^*\in\mathcal{H} : L_{\mathcal{D},f}\left(h^*\right)=0
    $$

    It is important to note that this condition also implies that $L_{\mathcal{S}}\left(h^*\right)=0$, because if it is valid for any set distributed according to $\mathcal{D}$, it will be even more so for the learning set $\mathcal{S}$ from which the algorithm learns.

    This last hypothesis is very important, because if the points of the learning set are correlated in some way with each other, this results in an obstacle to the search for an algorithm that finds the right pattern in the data. We indicate this statement with $\mathcal{S}\sim\mathcal{D}^m$, where $m$ is the number of points extracted from $\mathcal{S}$.
