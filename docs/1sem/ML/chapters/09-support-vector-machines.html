10  Support Vector Machines

10.1 Classification Margin

In Figure (fig:predictor-lineari-perceptron-soluzioni-diverse?), we mentioned that if a dataset is linearly separable, there will be multiple halfspaces capable of producing the correct solution to the classification problem.

Let’s now try to address the problem by asking: which is the best solution? Intuitively, we consider “better” those halfspaces that are able to separate the data “as much as possible,” that is, with a sufficiently large margin. We can then formally define the so-called classification margin, corresponding to the minimum distance between the halfspace and an element in the learning set.

In practice, the reason we want a larger margin is because such models turn out to be more “tolerant” of noise as we described it in the logistic regression algorithm (section (sec:regressione-logistica?)).

Let’s start from the definitions we already know. Given a learning set \(\mathcal{S}=\left\{\left(\boldsymbol{x_1},y_1\right),\dots,\left(\boldsymbol{x_n},y_n\right)\right\}\), it is said to be linearly separable if there exists a halfspace \((\boldsymbol{w},b)\) such that

\[ y_i\left(\sprod{\boldsymbol{x_i}}{\boldsymbol{w}}+b\right)>0 \hspace{0.65cm} \forall i \]

The separating hyperplane can be defined as

\[ L=\left\{\boldsymbol{v}:\sprod{v}{w}+b=0\right\} \]

The distance of a point \(\boldsymbol{x}\) from the hyperplane \(L\) is given by

\[ d(\boldsymbol{x},L)=\min(\norm{\boldsymbol{x}-\boldsymbol{v}}:\boldsymbol{v}\in L) \]

In other words, we consider the point \(\boldsymbol{v}\) on the hyperplane that minimizes the norm of the difference between \(\boldsymbol{x}\) and \(\boldsymbol{v}\) itself, and this norm corresponds precisely to the distance. The point (or points) \(\boldsymbol{x}\) that minimize the distance are called support vectors, as they “support” (in the sense of defining) the margin.

With the following theorem, we will make explicit the form of the distance \(d\) from a point to a hyperplane.

Assuming that \(\norm{\boldsymbol{w}}=1\), the distance between a point \(\boldsymbol{x}\) and a hyperplane \(L\) defined by \((\boldsymbol{w},b)\) is equal to

\[ d(\boldsymbol{x},L)=\abs{\sprod{\boldsymbol{x}}{\boldsymbol{w}}+b} \]

Let’s define the point \(\boldsymbol{z}\) as

\[ \boldsymbol{z}=\boldsymbol{x}-(\sprod{x}{w}+b)\boldsymbol{w} \]

We can easily show that this point belongs to the separating hyperplane:

\[\begin{align*} \sprod{z}{w}+b&=\sprod{x}{w}-(\sprod{x}{w}+b)\sprod{w}{w}+b=\\ &=\sprod{x}{w}-(\sprod{x}{w}+b)\norm{w}^2+b=\sprod{x}{w}-\sprod{x}{w}-b+b=0 \end{align*}\]

since \(\norm{\boldsymbol{w}}=1\).

The distance of \(\boldsymbol{z}\) from \(\boldsymbol{x}\) is

\[\begin{align*} \norm{\boldsymbol{x}-\boldsymbol{z}}&=\norm{\cancel{\boldsymbol{x}}-\cancel{\boldsymbol{x}}+(\sprod{x}{w}+b)\boldsymbol{w}}=\\ &=\abs{(\sprod{x}{w}+b)}\norm{\boldsymbol{w}}=\abs{(\sprod{x}{w}+b)} \end{align*}\]

Obviously, we do not know if \(\boldsymbol{z}\) is such as to minimize the distance between \(\boldsymbol{x}\) and the hyperplane, in the sense that \(\boldsymbol{z}\) is the closest point to \(\boldsymbol{x}\); what we know is that, in the “worst” case, the distance would be at most \(\norm{\boldsymbol{x}-\boldsymbol{z}}\), in the sense that \(\boldsymbol{z}\) would be the point furthest from \(\boldsymbol{x}\).

Let’s now consider a generic point \(\boldsymbol{z'}\) belonging to the hyperplane and let’s make its squared distance from \(\boldsymbol{x}\) explicit by adding and subtracting \(\boldsymbol{z}\):

\[ \norm{\boldsymbol{x}-\boldsymbol{z'}}^2=\norm{\boldsymbol{x}-\boldsymbol{z}+\boldsymbol{z}-\boldsymbol{z'}}^2=\norm{\boldsymbol{x}-\boldsymbol{z}}^2+\norm{\boldsymbol{z}-\boldsymbol{z'}}^2+2\left\langle\boldsymbol{x}-\boldsymbol{z},\boldsymbol{z}-\boldsymbol{z'}\right\rangle \]

Since \(\norm{\boldsymbol{z}-\boldsymbol{z'}}\) is certainly a non-negative quantity:

\[ \norm{\boldsymbol{x}-\boldsymbol{z'}}^2\geqslant\norm{\boldsymbol{x}-\boldsymbol{z}}^2+2\left\langle\boldsymbol{x}-\boldsymbol{z},\boldsymbol{z}-\boldsymbol{z'}\right\rangle \]

Let’s make \(\boldsymbol{z}\) explicit in the first term of the scalar product:

\[\begin{align*} \norm{\boldsymbol{x}-\boldsymbol{z'}}^2&\geqslant\norm{\boldsymbol{x}-\boldsymbol{z}}^2+2\left\langle\cancel{\boldsymbol{x}}-\cancel{\boldsymbol{x}}+(\sprod{x}{w}+b)\boldsymbol{w},\boldsymbol{z}-\boldsymbol{z'}\right\rangle=\\ &=\norm{\boldsymbol{x}-\boldsymbol{z}}^2+2\abs{\sprod{x}{w}+b}\left\langle\boldsymbol{w},\boldsymbol{z}-\boldsymbol{z'}\right\rangle \end{align*}\]

We know that \(\boldsymbol{w}\) has a direction orthogonal to the hyperplane, while \(\boldsymbol{z}-\boldsymbol{z'}\) will have a parallel direction, since they are two points located on the same hyperplane. It follows that the scalar product between them will be zero, therefore:

\[ \norm{\boldsymbol{x}-\boldsymbol{z'}}^2\geqslant\norm{\boldsymbol{x}-\boldsymbol{z}}^2 \]

We have thus come to the conclusion that a generic point \(\boldsymbol{z'}\) located on the hyperplane will have a greater distance from \(\boldsymbol{x}\) than \(\boldsymbol{z}\), also located on the hyperplane. It follows that \(\boldsymbol{z}\) is that point such as to guarantee the smallest distance, therefore from the calculated norm \(\norm{\boldsymbol{x}-\boldsymbol{z}}\) the thesis follows.

10.2 Hard-SVM

According to what we have seen from a computational point of view, the classification problem, assuming the data set is linearly separable, has the solution

\[ \argmax_{(\boldsymbol{w},b):\norm{\boldsymbol{w}}=1}\Big(\min_i\abs{\sprod{x_i}{w}+b}\Big):y_i\left(\sprod{x_i}{w}+b\right)>0 \hspace{0.65cm} \forall i \]

We can rewrite it in an equivalent but more compact form; in fact, since \(\abs{y_i}=1\), then we can remove the absolute value and simply write:

\[ \boxed{\argmax_{(\boldsymbol{w},b):\norm{\boldsymbol{w}}=1}\min_i\left[y_i\left(\sprod{x_i}{w}+b\right)\right]} \]

10.2.1 Formulation according to quadratic programming

We are now looking for an algorithm capable of finding the result of (eq:support-vector-machine-hard-svm-soluzione?). What is convenient to do is to try a different formulation, which leads the problem to be expressed in a form for which the algorithm for the solution is known.

First, we can write a point \(\boldsymbol{x}\) as a function of its orthogonal projection \(\boldsymbol{x_p}\) with respect to the separating hyperplane:

\[ \boldsymbol{x}=\boldsymbol{x_p}+\gamma\frac{\boldsymbol{w}}{\norm{\boldsymbol{w}}} \]

This writing is to be understood in the following way: to the point \(\boldsymbol{x_p}\) we add a “piece” in the same direction \(\boldsymbol{w}\), so we take the unit vector \(\frac{\boldsymbol{w}}{\norm{\boldsymbol{w}}}\) and add it \(\gamma\) times.

Since \(\boldsymbol{x_p}\) belongs to the hyperplane:

\[ \sprod{x_p}{w}+b=0 \]

Substituting \(\boldsymbol{x_p}\) from (support-vector-machine-formulazione-quadratica-1?) we can calculate \(\gamma\):

\[ \left\langle\boldsymbol{x}-\gamma\frac{\boldsymbol{w}}{\norm{\boldsymbol{w}}},\boldsymbol{w}\right\rangle+b=0 \ \Longrightarrow \ \sprod{x}{w}-\gamma\frac{\norm{\boldsymbol{w}}^{\cancel{2}}}{\cancel{\norm{\boldsymbol{w}}}}+b=0 \]

\[ \gamma=\frac{1}{\norm{\boldsymbol{w}}}\left(\sprod{x}{w}+b\right) \]

This quantity measures the distance between \(\boldsymbol{x}\) and the hyperplane, in fact:

\[ \boldsymbol{x}-\boldsymbol{x_p}=\gamma\frac{\boldsymbol{w}}{\norm{\boldsymbol{w}}} \ \Longrightarrow \ \norm{\boldsymbol{x}-\boldsymbol{x_p}}=\abs{\gamma}\frac{\cancel{\norm{\boldsymbol{w}}}}{\cancel{\norm{\boldsymbol{w}}}}=\frac{1}{\norm{\boldsymbol{w}}}\abs{\sprod{x}{w}+b} \]

which is not by chance the same result found in the previous theorem, since there \(\norm{\boldsymbol{w}}=1\).

Another way to maximize the distance, therefore, is to bring \(\abs{\sprod{x}{w}+b}\) as close as possible to \(1\) and minimize \(\norm{\boldsymbol{w}}\) as much as possible. Since we have already said that \(\abs{\sprod{x}{w}+b}\) is equivalent to \(y_i\left(\sprod{x}{w}+b\right)\), we can write the problem as

\[ \boxed{\argmin_{(\boldsymbol{w},b)}\norm{\boldsymbol{w}}^2 \ \text{with} \ \hspace{0.35cm} y_i\left(\sprod{x_i}{w}+b\right)\geqslant1 \ \hspace{0.35cm} \forall i} \]

The reason why \(\norm{\boldsymbol{w}}^2\) is chosen instead of \(\norm{\boldsymbol{w}}\) is that convex functions are preferred, as convergence to a solution is easier. The one exposed in (eq:support-vector-machine-formulazione-quadratica-problema?) is a problem of quadratic programming, for which solution algorithms are known. Once the solution \(\left(\boldsymbol{w_0},b_0\right)\) is found, the final output will consist of this quantity normalized with respect to \(\norm{\boldsymbol{w_0}}\):

\[ \boldsymbol{\hat{w}}=\frac{\boldsymbol{w_0}}{\norm{\boldsymbol{w_0}}} \ \hspace{0.75cm} \hat{b}=\frac{b_0}{\norm{\boldsymbol{w_0}}} \]

From a rigorous point of view, we can demonstrate the equivalence between (eq:support-vector-machine-hard-svm-soluzione?) and (eq:support-vector-machine-formulazione-quadratica-problema?).

The solution \(\left(\boldsymbol{w_0},b_0\right)\) to problem (eq:support-vector-machine-formulazione-quadratica-problema?) normalized with respect to \(\norm{\boldsymbol{w_0}}\) is also a solution to problem (eq:support-vector-machine-hard-svm-soluzione?).

Let \(\left(\boldsymbol{w^*},b^*\right)\) be a solution of the Hard-SVM. Let’s define \(\gamma^*\) as the margin of \(\left(\boldsymbol{w^*},b^*\right)\):

\[ \gamma^*=\min_i\left[y_i\left(\sprod{x_i}{w^*}+b^*\right)\right] \]

From this expression it follows that

\[ y_i\left(\sprod{x_i}{w^*}+b^*\right)\geqslant\gamma^* \]

Let’s divide both members by \(\gamma^*\):

\[ y_i\left(\left\langle\boldsymbol{x_i},\frac{\boldsymbol{w}^*}{\gamma^*}\right\rangle+\frac{b^*}{\gamma^*}\right)\geqslant1 \]

We therefore find a hyperplane \(\left(\frac{\boldsymbol{w^*}}{\gamma^*},\frac{b^*}{\gamma^*}\right)\) that respects the condition of (eq:support-vector-machine-formulazione-quadratica-problema?). At this point there are two possibilities: either \(\boldsymbol{w^*}\) is also the vector whose squared norm is the smallest, for which we would have \(\boldsymbol{w_0}=\frac{\boldsymbol{w^*}}{\gamma^*}\), or the vector with the smallest squared norm is another one, and therefore \(\boldsymbol{w_0}\neq\frac{\boldsymbol{w^*}}{\gamma^*}\). We can therefore state that

\[ \norm{\boldsymbol{w_0}}^2\leqslant\norm{\frac{\boldsymbol{w^*}}{\gamma^*}}^2 \ \Longrightarrow \ \norm{\boldsymbol{w_0}}\leqslant\norm{\frac{\boldsymbol{w^*}}{\gamma^*}}=\frac{1}{\abs{\gamma^*}} \]

since \(\norm{\boldsymbol{w^*}}=1\), being a solution of the Hard-SVM.

We can also state that \(\gamma^*\) is a certainly positive quantity, because since \(\left(\boldsymbol{w^*},b^*\right)\) is a solution of the Hard-SVM, then it will classify all points correctly, therefore \(y_i\) and \(\sprod{x_i}{w^*}+b^*\) will always have the same sign and their product will be positive. It follows that

\[ \norm{\boldsymbol{w_0}}\leqslant\frac{1}{\gamma^*} \]

Here the first part of the proof ends. Let’s now move on to the second part, in which the objective is to show that the output obtained from quadratic programming, that is \(\left(\boldsymbol{\hat{w}},\hat{b}\right)=\frac{1}{\norm{\boldsymbol{w_0}}}\left(\boldsymbol{w_0},b_0\right)\), is a hyperplane with a margin greater than or equal to the one associated with \(\left(\boldsymbol{w^*},b^*\right)\).

Let’s start by making explicit the classification of the \(i\)-th point:

\[ y_i\left(\sprod{x_i}{\hat{w}}+\hat{b}\right)=\frac{1}{\norm{\boldsymbol{w_0}}}y_i\left(\sprod{x_i}{w_0}+b_0\right) \]

Since \(\left(\boldsymbol{w_0},b_0\right)\) is a solution of (eq:support-vector-machine-formulazione-quadratica-problema?), then \(y_i\left(\sprod{x_i}{w_0}+b_0\right)\geqslant1\) for every point, therefore

\[ y_i\left(\sprod{x_i}{\hat{w}}+\hat{b}\right)\geqslant\frac{1}{\norm{\boldsymbol{w_0}}} \ \hspace{0.65cm} \forall i \]

From (eq:support-vector-machine-hard-svm-teorema-1?) we find another lower bound:

\[ \frac{1}{\norm{\boldsymbol{w_0}}}\geqslant\gamma^* \ \Longrightarrow \ y_i\left(\sprod{x_i}{\hat{w}}+\hat{b}\right)\geqslant\gamma^* \ \hspace{0.65cm} \forall i \]

We recall that \(\gamma^*\) is the margin relative to \(\left(\boldsymbol{w^*},b^*\right)\), which is a solution of (eq:support-vector-machine-hard-svm-soluzione?). We have thus found that the solution to problem (eq:support-vector-machine-formulazione-quadratica-problema?), that is \(\left(\boldsymbol{\hat{w}},\hat{b}\right)\), is also a solution to problem (eq:support-vector-machine-hard-svm-soluzione?), since:

- $\norm{\boldsymbol{\hat{w}}}=1$
- $\left(\boldsymbol{\hat{w}},\hat{b}\right)$ correctly classifies all points
- $\left(\boldsymbol{\hat{w}},\hat{b}\right)$ has a margin at least as large as that of $\left(\boldsymbol{w^*},b^*\right)$

10.2.1.1 Representation in homogeneous coordinates

In paragraph (sec:funzioni-lineari-omogenee?) we discussed the possibility of applying the transformation of homogeneous coordinates, adding a dimension to both the vector \(\boldsymbol{x}\) and the vector \(\boldsymbol{w}\): in the first case it will be the value \(1\) added as the first element, while in the second case it will be \(b\) added always as the first element. We will therefore perform the step

\[ \sprod{x}{w}+b \ \longrightarrow \ \sprod{x'}{w'} \]

In the case where this transformation is applied, (eq:support-vector-machine-formulazione-quadratica-problema?) would become:

\[ \argmin_{\boldsymbol{w'}}\norm{\boldsymbol{w'}}^2 \ \hspace{0.65cm} \text{with} \ \hspace{0.35cm} y_i\sprod{x'}{w'}\geqslant1 \ \hspace{0.35cm} \forall i \]

If we look closely, this problem is not exactly identical to the previous one: the bias, in fact, is this time included in the process of minimizing the weight vector. Formally, therefore, they are two different problems, although in practice the inclusion of \(b\) in the process does not make a real difference, so the solution for one problem is very often valid for the other as well. 10.2.2 Fritz John conditions

The formulation according to quadratic programming is useful for identifying an interesting property related to support vectors. It is expressed in the following theorem.

Let \(\boldsymbol{w'}\) be a weight vector subject to what is written in equation (eq:support-vector-machine-formulazione-quadratica-problema?). Let’s define the set \(I\) as

\[ I=\left\{i:\abs{\sprod{w'}{x_i}}=1\right\} \]

If the set \(I\) is composed of \(m\) elements, then there exists a set of coefficients \(\alpha_1,\alpha_2,\dots,\alpha_m\) such that

\[ \boldsymbol{w'}=\sum_{i\in I}\alpha_i\boldsymbol{x_i} \]

The theorem therefore states that, as long as the vector \(\boldsymbol{w'}\) respects the conditions of the problem, it can be expressed as a linear combination of the only support vectors, since they are the vectors for which the scalar product with \(\boldsymbol{w'}\) is the smallest possible, that is \(1\).

The proof of this theorem is not required, but it is useful to know that it is proved through a lemma that uses the so-called Fritz John conditions, and by combining this lemma with equation (eq:support-vector-machine-formulazione-quadratica-problema?) we obtain precisely the theorem just written.

10.2.3 Duality

Every optimization problem can be expressed in two forms: the primal form and the dual form. The one of quadratic programming is an optimization problem, and it has been expressed in its main form, called precisely primal, in the context of support vectors. It follows, therefore, that there is also a dual form of the same problem, which is shown to be

\[ \max_{\alpha\in\mathbb{R}^m:\alpha\geqslant0}\sum_{i=1}^m\alpha_i-\frac{1}{2}\sum_{i=1}^m\sum_{j=1}^m\alpha_i\alpha_jy_iy_j\sprod{x_i}{x_j} \]

We do not go into the detail of how to derive this formula; suffice it to say that the \(\alpha_i\) are the so-called Lagrange multipliers, useful for determining which constraints are active and to what extent they contribute to the construction of the optimal hyperplane.

The most important aspect of the formula just written is the one related to the scalar product between vectors: according to this formulation, in fact, it is not necessary to have access to the vectors themselves, but only to the scalar products between them. This may not seem like a great advantage now, but we will see how it will become one when in the next chapter we talk about the kernel trick.

10.3 Soft-SVM

The main problem with Hard-SVM is that it is only applicable in cases where the dataset is linearly separable, and this is also the reason for the term hard in its name, imposing this “rigid” constraint. As we have already reiterated in the past, in real problems the property of linear separability is rarely respected, but there are some outliers that do not allow for a clear and defined separation. We will therefore need to use an approach that can tolerate some errors, and this will be called soft-SVM.

For this new approach, we will recycle the idea of regularization that we introduced in section (sec:regolarizzazione?). First of all, we define variables \(\xi_i\) called slack (The term slack* translated means loosening, to symbolize the relaxation of the constraints with respect to the hard-SVM.)* which are added to the second member of the constraint of quadratic programming in the following way:

\[ y_i\left(\sprod{x_i}{w}+b\right)\geqslant1*-\xi_i* \]

Intuitively, \(\xi_i\) is a measure of how much the constraint is violated:

  • If \(0<\xi_i<1\), then the \(i\)-th point is classified correctly, but is within the margin.
  • If \(1<\xi_i<2\), then the \(i\)-th point is misclassified and is within the margin.
  • If \(\xi_i>2\), then the \(i\)-th point is misclassified and is beyond the margin.

The objective will therefore be to minimize the sum of \(\norm{\boldsymbol{w}}^2\) and the average of the stack variables. We understand then that the squared norm of the weight vector will assume the role of a stabilizer, therefore it will be multiplied by a parameter \(\lambda\) that will regulate the fitting-stability trade-off:

\[ \boxed{\min_{\boldsymbol{w},b,\xi}\left(\frac{1}{m}\sum_{i=1}^m\xi_i+\lambda\norm{\boldsymbol{w}}^2\right) \ \hspace{0.65cm} \text{with} \ \hspace{0.35cm} y_i\left(\sprod{x_i}{w}+b\right)\geqslant1-\xi_i,\xi_i\geqslant0 \ \hspace{0.35cm} \forall i} \]

Let’s pay attention to the fact that here it is not about the \(\argmin\) function but about the \(\min\) function, as the objective is to minimize the sum itself also with respect to all the slack variables \(\xi_i\) which are not provided for in the final output: the latter, in fact, will simply be equal to \((\boldsymbol{w},b)\). (This does not explain, however, why we include \(b\) in the search for the minimum if no \(b\) is present in the expression; although I do not know how to justify it, I report it anyway because maybe I’m missing something.)

Let’s see how the result changes as a function of \(\lambda\): (The professor tried to explain the two cases, getting confused three times in two minutes, and still ending up saying something different from what was written on the slide. I have tried to explain the two extremes in my own words as I understand the situations.)

  • \(\boldsymbol{\lambda}\) large: the term \(\lambda\norm{\boldsymbol{w}}^2\) will dominate the function, so the model will give great importance to keeping \(\norm{\boldsymbol{w}}\) small, tending to choose a vector \(\boldsymbol{w}\) with a very low norm. This leads to a wider margin between the classes at the cost of increasing the number of classification errors (expressed with higher values of \(\xi_i\)). Due to the underestimation of errors, the model could be subject to underfitting.

  • \(\boldsymbol{\lambda}\) small: the error penalty term \(\frac{1}{m}\sum\limits_{i=1}^m\xi_i\) will dominate the function, so the model will give more weight to minimizing classification errors rather than maintaining a large margin. This means that the model will be willing to tolerate larger values of \(\norm{\boldsymbol{w}}\) in order to minimize the \(\xi_i\). Since the model could overfit the data to minimize the margin, there is a risk of overfitting.

10.3.1 Reformulation with the hinge loss

Since we have rewritten the function with the stabilizer, we can try to rewrite the other piece in the form of a loss function. We would therefore want something like

\[ \min_{\boldsymbol{w},b}\left(L_{\mathcal{S}}(\boldsymbol{w},b)+\lambda\norm{\boldsymbol{w}}^2\right)=\min_{\boldsymbol{w},b}\left(\frac{1}{m}\sum_{i=1}^m\ell\left((\boldsymbol{w},b),\left(\boldsymbol{x_i},y_i\right)\right)+\lambda\norm{\boldsymbol{w}}^2\right) \]

Let’s go back to the inequality (eq:support-vector-machine-soft-svm-variabili-slack?) and consider the extreme case of equality. Isolating \(\xi_i\) we get:

\[ \xi_i=1-y_i\left(\sprod{x_i}{w}+b\right) \]

Since if the error is classified correctly we want the loss to be zero, it will result:

\[ \ell^{hinge}\left((\boldsymbol{w},b),(\boldsymbol{x},y)\right)=\max\left(0,1-y(\sprod{x}{w}+b)\right) \]

This loss function, as it is written, is called hinge loss

For correctly classified points, we have that \(y(\sprod{x}{w}+b)>0\): in the case where \(y(\sprod{x}{w}+b)>1\), they are both correctly classified and outside the margin, and in fact it results that \(1-y(\sprod{x}{w}+b)<0\), from which it follows that \(l^{hinge}=0\); in the case where \(0<y(\sprod{x}{w}+b)<1\), the point is correctly classified but within the margin, and in fact the error becomes greater than zero, being equal to \(0<\ell^{hinge}<1\). The last possibility is that \(y(\sprod{x}{w}+b)<0\), a condition that occurs when the points are not correctly classified, and results in an even greater loss: \(l^{hinge}>1\).

We therefore find the following formulation of the problem:

\[ \boxed{\min_{\boldsymbol{w},b}\left(\frac{1}{m}\sum_{i=1}^m\max\left(0,1-y_i\left(\sprod{x_i}{w}+b\right)\right)+\lambda\norm{\boldsymbol{w}}^2\right)} \]

We can formally prove that expressions (eq:support-vector-machine-soft-svm-prima-formulazione?) and (eq:support-vector-machine-soft-svm-problema-hinge-loss?) are equivalent.

The expressions (eq:support-vector-machine-soft-svm-prima-formulazione?) and (eq:support-vector-machine-soft-svm-problema-hinge-loss?) are equivalent.

Let’s start from (eq:support-vector-machine-soft-svm-prima-formulazione?) and calculate the smallest values of \(\xi_i\) for which the inequality is valid. First, let’s express the inequality as a function of \(\xi_i\):

\[ \xi_i\geqslant1-y_i\left(\sprod{x_i}{w}+b\right) \]

We distinguish two cases:

- $y_i\left(\sprod{x_i}{w}+b\right)\geqslant1$

This condition implies that $1-y_i\left(\sprod{x_i}{w}+b\right)\leqslant0$, so on the right-hand side of @eq:support-vector-machine-soft-svm-hinge-loss-teorema we have a negative value. Recalling that the other constraint is $\xi_i\geqslant0$, then the smallest value for which the inequality is valid is $\xi_i=0$.

- $y_i\left(\sprod{x_i}{w}+b\right)<1$

This condition implies that $1-y_i\left(\sprod{x_i}{w}+b\right)>0$, so on the right-hand side of @eq:support-vector-machine-soft-svm-hinge-loss-teorema we have a positive value. The smallest value for which @eq:support-vector-machine-soft-svm-hinge-loss-teorema is valid is therefore the one obtained from the case of equality, that is $\xi_i=1-y_i\left(\sprod{x_i}{w}+b\right)$.

We have identified the following function:

\[ \xi_i= \begin{cases} 1-y_i\left(\sprod{x_i}{w}+b\right) \ \ &y_i\left(\sprod{x_i}{w}+b\right)<1\\ 0 \ \ &y_i\left(\sprod{x_i}{w}+b\right)\geqslant1 \end{cases} \]

We can rewrite it by bringing all the conditions on the right to the first member:

\[ \xi_i= \begin{cases} 0 \ \ &1-y_i\left(\sprod{x_i}{w}+b\right)\leqslant0\\ 1-y_i\left(\sprod{x_i}{w}+b\right) \ \ &1-y_i\left(\sprod{x_i}{w}+b\right)>0 \end{cases} \]

From the latter it clearly emerges that

\[ \xi_i=\max\left(0, 1-y_i\left(\sprod{x_i}{w}+b\right)\right) \]

We can substitute this expression in (eq:support-vector-machine-soft-svm-prima-formulazione?), omitting the conditions, as they are already included as we have seen. By doing so, we obtain exactly (eq:support-vector-machine-soft-svm-problema-hinge-loss?), thus proving the thesis.

Whether one formulation or the other is considered, it is in any case a minimization problem, which is usually approached iteratively through an algorithm that we will see later, the Stochastic Gradient Descent.

## Classification Margin

In Figure @fig:predictor-lineari-perceptron-soluzioni-diverse, we mentioned that if a dataset is linearly separable, there will be multiple halfspaces capable of producing the correct solution to the classification problem.

Let's now try to address the problem by asking: which is the best solution? Intuitively, we consider "better" those halfspaces that are able to separate the data "as much as possible," that is, with a sufficiently large margin. We can then formally define the so-called **classification margin**, corresponding to the minimum distance between the halfspace and an element in the learning set.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.75\textwidth]{Grafiche/Immagini/support-vector-machine-margine-di-classificazione.png}
    \caption{On the left, two examples of hyperplanes that correctly classify the points, while on the right, the one with the largest margin, considered the best solution.}
\end{figure}

In practice, the reason we want a larger margin is because such models turn out to be more "tolerant" of noise as we described it in the logistic regression algorithm (section @sec:regressione-logistica).

Let's start from the definitions we already know. Given a learning set $\mathcal{S}=\left\{\left(\boldsymbol{x_1},y_1\right),\dots,\left(\boldsymbol{x_n},y_n\right)\right\}$, it is said to be *linearly separable* if there exists a halfspace $(\boldsymbol{w},b)$ such that

$$
    y_i\left(\sprod{\boldsymbol{x_i}}{\boldsymbol{w}}+b\right)>0 \hspace{0.65cm} \forall i
$$

The separating hyperplane can be defined as

$$
    L=\left\{\boldsymbol{v}:\sprod{v}{w}+b=0\right\}
$$

The distance of a point $\boldsymbol{x}$ from the hyperplane $L$ is given by

$$
    d(\boldsymbol{x},L)=\min(\norm{\boldsymbol{x}-\boldsymbol{v}}:\boldsymbol{v}\in L)
$$

In other words, we consider the point $\boldsymbol{v}$ on the hyperplane that minimizes the norm of the difference between $\boldsymbol{x}$ and $\boldsymbol{v}$ itself, and this norm corresponds precisely to the distance. The point (or points) $\boldsymbol{x}$ that minimize the distance are called **support vectors**, as they "support" (in the sense of *defining*) the margin.

With the following theorem, we will make explicit the form of the distance $d$ from a point to a hyperplane.

::: {.theorem name="Theorem on the distance from a hyperplane"}
Assuming that $\norm{\boldsymbol{w}}=1$, the distance between a point $\boldsymbol{x}$ and a hyperplane $L$ defined by $(\boldsymbol{w},b)$ is equal to

$$
    d(\boldsymbol{x},L)=\abs{\sprod{\boldsymbol{x}}{\boldsymbol{w}}+b}
$$

\exrule

Let's define the point $\boldsymbol{z}$ as

$$
    \boldsymbol{z}=\boldsymbol{x}-(\sprod{x}{w}+b)\boldsymbol{w}
$$

We can easily show that this point belongs to the separating hyperplane:

\begin{align*}
    \sprod{z}{w}+b&=\sprod{x}{w}-(\sprod{x}{w}+b)\sprod{w}{w}+b=\\
    &=\sprod{x}{w}-(\sprod{x}{w}+b)\norm{w}^2+b=\sprod{x}{w}-\sprod{x}{w}-b+b=0
\end{align*}

since $\norm{\boldsymbol{w}}=1$.

The distance of $\boldsymbol{z}$ from $\boldsymbol{x}$ is

\begin{align*}
    \norm{\boldsymbol{x}-\boldsymbol{z}}&=\norm{\cancel{\boldsymbol{x}}-\cancel{\boldsymbol{x}}+(\sprod{x}{w}+b)\boldsymbol{w}}=\\
    &=\abs{(\sprod{x}{w}+b)}\norm{\boldsymbol{w}}=\abs{(\sprod{x}{w}+b)}
\end{align*}

Obviously, we do not know if $\boldsymbol{z}$ is such as to minimize the distance between $\boldsymbol{x}$ and the hyperplane, in the sense that $\boldsymbol{z}$ is the closest point to $\boldsymbol{x}$; what we know is that, in the "worst" case, the distance would be at most $\norm{\boldsymbol{x}-\boldsymbol{z}}$, in the sense that $\boldsymbol{z}$ would be the point furthest from $\boldsymbol{x}$.

Let's now consider a generic point $\boldsymbol{z'}$ belonging to the hyperplane and let's make its squared distance from $\boldsymbol{x}$ explicit by adding and subtracting $\boldsymbol{z}$:

$$
    \norm{\boldsymbol{x}-\boldsymbol{z'}}^2=\norm{\boldsymbol{x}-\boldsymbol{z}+\boldsymbol{z}-\boldsymbol{z'}}^2=\norm{\boldsymbol{x}-\boldsymbol{z}}^2+\norm{\boldsymbol{z}-\boldsymbol{z'}}^2+2\left\langle\boldsymbol{x}-\boldsymbol{z},\boldsymbol{z}-\boldsymbol{z'}\right\rangle
$$

Since $\norm{\boldsymbol{z}-\boldsymbol{z'}}$ is certainly a non-negative quantity:

$$
    \norm{\boldsymbol{x}-\boldsymbol{z'}}^2\geqslant\norm{\boldsymbol{x}-\boldsymbol{z}}^2+2\left\langle\boldsymbol{x}-\boldsymbol{z},\boldsymbol{z}-\boldsymbol{z'}\right\rangle
$$

Let's make $\boldsymbol{z}$ explicit in the first term of the scalar product:

\begin{align*}
    \norm{\boldsymbol{x}-\boldsymbol{z'}}^2&\geqslant\norm{\boldsymbol{x}-\boldsymbol{z}}^2+2\left\langle\cancel{\boldsymbol{x}}-\cancel{\boldsymbol{x}}+(\sprod{x}{w}+b)\boldsymbol{w},\boldsymbol{z}-\boldsymbol{z'}\right\rangle=\\
    &=\norm{\boldsymbol{x}-\boldsymbol{z}}^2+2\abs{\sprod{x}{w}+b}\left\langle\boldsymbol{w},\boldsymbol{z}-\boldsymbol{z'}\right\rangle
\end{align*}

We know that $\boldsymbol{w}$ has a direction orthogonal to the hyperplane, while $\boldsymbol{z}-\boldsymbol{z'}$ will have a parallel direction, since they are two points located on the same hyperplane. It follows that the scalar product between them will be zero, therefore:

$$
    \norm{\boldsymbol{x}-\boldsymbol{z'}}^2\geqslant\norm{\boldsymbol{x}-\boldsymbol{z}}^2
$$

We have thus come to the conclusion that a generic point $\boldsymbol{z'}$ located on the hyperplane will have a greater distance from $\boldsymbol{x}$ than $\boldsymbol{z}$, also located on the hyperplane. It follows that $\boldsymbol{z}$ is that point such as to guarantee the smallest distance, therefore from the calculated norm $\norm{\boldsymbol{x}-\boldsymbol{z}}$ the thesis follows.
:::

## Hard-SVM

According to what we have seen from a computational point of view, the classification problem, assuming the data set is linearly separable, has the solution

$$
    \argmax_{(\boldsymbol{w},b):\norm{\boldsymbol{w}}=1}\Big(\min_i\abs{\sprod{x_i}{w}+b}\Big):y_i\left(\sprod{x_i}{w}+b\right)>0 \hspace{0.65cm} \forall i
$$

We can rewrite it in an equivalent but more compact form; in fact, since $\abs{y_i}=1$, then we can remove the absolute value and simply write:

$$
    \boxed{\argmax_{(\boldsymbol{w},b):\norm{\boldsymbol{w}}=1}\min_i\left[y_i\left(\sprod{x_i}{w}+b\right)\right]}
$$

### Formulation according to quadratic programming

We are now looking for an algorithm capable of finding the result of @eq:support-vector-machine-hard-svm-soluzione. What is convenient to do is to try a different formulation, which leads the problem to be expressed in a form for which the algorithm for the solution is known.

First, we can write a point $\boldsymbol{x}$ as a function of its orthogonal projection $\boldsymbol{x_p}$ with respect to the separating hyperplane:

$$
    \boldsymbol{x}=\boldsymbol{x_p}+\gamma\frac{\boldsymbol{w}}{\norm{\boldsymbol{w}}}
$$

This writing is to be understood in the following way: to the point $\boldsymbol{x_p}$ we add a "piece" in the same direction $\boldsymbol{w}$, so we take the unit vector $\frac{\boldsymbol{w}}{\norm{\boldsymbol{w}}}$ and add it $\gamma$ times.

Since $\boldsymbol{x_p}$ belongs to the hyperplane:

$$
    \sprod{x_p}{w}+b=0
$$

Substituting $\boldsymbol{x_p}$ from @support-vector-machine-formulazione-quadratica-1 we can calculate $\gamma$:

$$
    \left\langle\boldsymbol{x}-\gamma\frac{\boldsymbol{w}}{\norm{\boldsymbol{w}}},\boldsymbol{w}\right\rangle+b=0 \ \Longrightarrow \ \sprod{x}{w}-\gamma\frac{\norm{\boldsymbol{w}}^{\cancel{2}}}{\cancel{\norm{\boldsymbol{w}}}}+b=0
$$

$$
    \gamma=\frac{1}{\norm{\boldsymbol{w}}}\left(\sprod{x}{w}+b\right)
$$

This quantity measures the distance between $\boldsymbol{x}$ and the hyperplane, in fact:

$$
    \boldsymbol{x}-\boldsymbol{x_p}=\gamma\frac{\boldsymbol{w}}{\norm{\boldsymbol{w}}} \ \Longrightarrow \ \norm{\boldsymbol{x}-\boldsymbol{x_p}}=\abs{\gamma}\frac{\cancel{\norm{\boldsymbol{w}}}}{\cancel{\norm{\boldsymbol{w}}}}=\frac{1}{\norm{\boldsymbol{w}}}\abs{\sprod{x}{w}+b}
$$

which is not by chance the same result found in the previous theorem, since there $\norm{\boldsymbol{w}}=1$.

Another way to maximize the distance, therefore, is to bring $\abs{\sprod{x}{w}+b}$ as close as possible to $1$ and minimize $\norm{\boldsymbol{w}}$ as much as possible. Since we have already said that $\abs{\sprod{x}{w}+b}$ is equivalent to $y_i\left(\sprod{x}{w}+b\right)$, we can write the problem as

$$
    \boxed{\argmin_{(\boldsymbol{w},b)}\norm{\boldsymbol{w}}^2 \ \text{with} \ \hspace{0.35cm} y_i\left(\sprod{x_i}{w}+b\right)\geqslant1 \ \hspace{0.35cm} \forall i}
$$

The reason why $\norm{\boldsymbol{w}}^2$ is chosen instead of $\norm{\boldsymbol{w}}$ is that *convex* functions are preferred, as convergence to a solution is easier. The one exposed in @eq:support-vector-machine-formulazione-quadratica-problema is a problem of **quadratic programming**, for which solution algorithms are known. Once the solution $\left(\boldsymbol{w_0},b_0\right)$ is found, the final output will consist of this quantity normalized with respect to $\norm{\boldsymbol{w_0}}$:

$$
    \boldsymbol{\hat{w}}=\frac{\boldsymbol{w_0}}{\norm{\boldsymbol{w_0}}} \ \hspace{0.75cm} \hat{b}=\frac{b_0}{\norm{\boldsymbol{w_0}}}
$$

From a rigorous point of view, we can demonstrate the equivalence between @eq:support-vector-machine-hard-svm-soluzione and @eq:support-vector-machine-formulazione-quadratica-problema.

::: {.theorem name="Equivalence between Hard-SVM and quadratic program"}
The solution $\left(\boldsymbol{w_0},b_0\right)$ to problem @eq:support-vector-machine-formulazione-quadratica-problema normalized with respect to $\norm{\boldsymbol{w_0}}$ is also a solution to problem @eq:support-vector-machine-hard-svm-soluzione.

\exrule

Let $\left(\boldsymbol{w^*},b^*\right)$ be a solution of the Hard-SVM. Let's define $\gamma^*$ as the margin of $\left(\boldsymbol{w^*},b^*\right)$:

$$
    \gamma^*=\min_i\left[y_i\left(\sprod{x_i}{w^*}+b^*\right)\right]
$$

From this expression it follows that

$$
    y_i\left(\sprod{x_i}{w^*}+b^*\right)\geqslant\gamma^*
$$

Let's divide both members by $\gamma^*$: 

$$
    y_i\left(\left\langle\boldsymbol{x_i},\frac{\boldsymbol{w}^*}{\gamma^*}\right\rangle+\frac{b^*}{\gamma^*}\right)\geqslant1
$$

We therefore find a hyperplane $\left(\frac{\boldsymbol{w^*}}{\gamma^*},\frac{b^*}{\gamma^*}\right)$ that respects the condition of @eq:support-vector-machine-formulazione-quadratica-problema. At this point there are two possibilities: either $\boldsymbol{w^*}$ is also the vector whose squared norm is the smallest, for which we would have $\boldsymbol{w_0}=\frac{\boldsymbol{w^*}}{\gamma^*}$, or the vector with the smallest squared norm is another one, and therefore $\boldsymbol{w_0}\neq\frac{\boldsymbol{w^*}}{\gamma^*}$. We can therefore state that

$$
    \norm{\boldsymbol{w_0}}^2\leqslant\norm{\frac{\boldsymbol{w^*}}{\gamma^*}}^2 \ \Longrightarrow \ \norm{\boldsymbol{w_0}}\leqslant\norm{\frac{\boldsymbol{w^*}}{\gamma^*}}=\frac{1}{\abs{\gamma^*}}
$$

since $\norm{\boldsymbol{w^*}}=1$, being a solution of the Hard-SVM.

We can also state that $\gamma^*$ is a certainly positive quantity, because since $\left(\boldsymbol{w^*},b^*\right)$ is a solution of the Hard-SVM, then it will classify all points correctly, therefore $y_i$ and $\sprod{x_i}{w^*}+b^*$ will always have the same sign and their product will be positive. It follows that

$$
    \norm{\boldsymbol{w_0}}\leqslant\frac{1}{\gamma^*}
$$

Here the first part of the proof ends. Let's now move on to the second part, in which the objective is to show that the output obtained from quadratic programming, that is $\left(\boldsymbol{\hat{w}},\hat{b}\right)=\frac{1}{\norm{\boldsymbol{w_0}}}\left(\boldsymbol{w_0},b_0\right)$, is a hyperplane with a margin greater than or equal to the one associated with $\left(\boldsymbol{w^*},b^*\right)$.

Let's start by making explicit the classification of the $i$-th point:

$$
    y_i\left(\sprod{x_i}{\hat{w}}+\hat{b}\right)=\frac{1}{\norm{\boldsymbol{w_0}}}y_i\left(\sprod{x_i}{w_0}+b_0\right)
$$

Since $\left(\boldsymbol{w_0},b_0\right)$ is a solution of @eq:support-vector-machine-formulazione-quadratica-problema, then $y_i\left(\sprod{x_i}{w_0}+b_0\right)\geqslant1$ for *every* point, therefore

$$
    y_i\left(\sprod{x_i}{\hat{w}}+\hat{b}\right)\geqslant\frac{1}{\norm{\boldsymbol{w_0}}} \ \hspace{0.65cm} \forall i
$$

From @eq:support-vector-machine-hard-svm-teorema-1 we find another lower bound:

$$
    \frac{1}{\norm{\boldsymbol{w_0}}}\geqslant\gamma^* \ \Longrightarrow \ y_i\left(\sprod{x_i}{\hat{w}}+\hat{b}\right)\geqslant\gamma^* \ \hspace{0.65cm} \forall i
$$

We recall that $\gamma^*$ is the margin relative to $\left(\boldsymbol{w^*},b^*\right)$, which is a solution of @eq:support-vector-machine-hard-svm-soluzione. We have thus found that the solution to problem @eq:support-vector-machine-formulazione-quadratica-problema, that is $\left(\boldsymbol{\hat{w}},\hat{b}\right)$, is also a solution to problem @eq:support-vector-machine-hard-svm-soluzione, since:


    - $\norm{\boldsymbol{\hat{w}}}=1$
    - $\left(\boldsymbol{\hat{w}},\hat{b}\right)$ correctly classifies all points
    - $\left(\boldsymbol{\hat{w}},\hat{b}\right)$ has a margin at least as large as that of $\left(\boldsymbol{w^*},b^*\right)$

:::

#### Representation in homogeneous coordinates

In paragraph @sec:funzioni-lineari-omogenee we discussed the possibility of applying the transformation of homogeneous coordinates, adding a dimension to both the vector $\boldsymbol{x}$ and the vector $\boldsymbol{w}$: in the first case it will be the value $1$ added as the first element, while in the second case it will be $b$ added always as the first element. We will therefore perform the step

$$
    \sprod{x}{w}+b \ \longrightarrow \ \sprod{x'}{w'}
$$

In the case where this transformation is applied, @eq:support-vector-machine-formulazione-quadratica-problema would become:

$$
    \argmin_{\boldsymbol{w'}}\norm{\boldsymbol{w'}}^2 \ \hspace{0.65cm} \text{with} \ \hspace{0.35cm} y_i\sprod{x'}{w'}\geqslant1 \ \hspace{0.35cm} \forall i
$$

If we look closely, this problem is not exactly identical to the previous one: the bias, in fact, is this time *included* in the process of minimizing the weight vector. Formally, therefore, they are two different problems, although in practice the inclusion of $b$ in the process does not make a real difference, so the solution for one problem is very often valid for the other as well. *(The course is taught by two engineers. Expecting a higher level of detail than this is substantial utopia.)*

### Fritz John conditions

The formulation according to quadratic programming is useful for identifying an interesting property related to support vectors. It is expressed in the following theorem.

::: {.theorem name="Algebraic property of support vectors"}
Let $\boldsymbol{w'}$ be a weight vector subject to what is written in equation @eq:support-vector-machine-formulazione-quadratica-problema. Let's define the set $I$ as

$$
    I=\left\{i:\abs{\sprod{w'}{x_i}}=1\right\}
$$

If the set $I$ is composed of $m$ elements, then there exists a set of coefficients $\alpha_1,\alpha_2,\dots,\alpha_m$ such that

$$
    \boldsymbol{w'}=\sum_{i\in I}\alpha_i\boldsymbol{x_i}
$$
:::

The theorem therefore states that, as long as the vector $\boldsymbol{w'}$ respects the conditions of the problem, it can be expressed as a linear combination of the *only* support vectors, since they are the vectors for which the scalar product with $\boldsymbol{w'}$ is the smallest possible, that is $1$.

The proof of this theorem is not required, but it is useful to know that it is proved through a lemma that uses the so-called *Fritz John conditions*, and by combining this lemma with equation @eq:support-vector-machine-formulazione-quadratica-problema we obtain precisely the theorem just written.

### Duality

Every optimization problem can be expressed in two forms: the *primal form* and the *dual form*. The one of quadratic programming is an optimization problem, and it has been expressed in its main form, called precisely *primal*, in the context of support vectors. It follows, therefore, that there is also a dual form of the same problem, which is shown to be

$$
    \max_{\alpha\in\mathbb{R}^m:\alpha\geqslant0}\sum_{i=1}^m\alpha_i-\frac{1}{2}\sum_{i=1}^m\sum_{j=1}^m\alpha_i\alpha_jy_iy_j\sprod{x_i}{x_j}
$$

We do not go into the detail of how to derive this formula; suffice it to say that the $\alpha_i$ are the so-called *Lagrange multipliers*, useful for determining which constraints are active and to what extent they contribute to the construction of the optimal hyperplane.

The most important aspect of the formula just written is the one related to the scalar product between vectors: according to this formulation, in fact, it is not necessary to have access to the vectors themselves, but only to the scalar products between them. This may not seem like a great advantage now, but we will see how it will become one when in the next chapter we talk about the *kernel trick*.

## Soft-SVM

The main problem with Hard-SVM is that it is only applicable in cases where the dataset is linearly separable, and this is also the reason for the term *hard* in its name, imposing this "rigid" constraint. As we have already reiterated in the past, in real problems the property of linear separability is rarely respected, but there are some outliers that do not allow for a clear and defined separation. We will therefore need to use an approach that can tolerate some errors, and this will be called **soft-SVM**.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.83\textwidth]{Grafiche/Immagini/support-vector-machine-soft-svm-esempio.jpg}
    \caption{On the left an example of hard-SVM, on the right an example of soft-SVM}
\end{figure}

For this new approach, we will recycle the idea of regularization that we introduced in section @sec:regolarizzazione. First of all, we define variables $\xi_i$ called *slack* *(The term *slack* translated means *loosening*, to symbolize the relaxation of the constraints with respect to the hard-SVM.)* which are added to the second member of the constraint of quadratic programming in the following way:

$$
    y_i\left(\sprod{x_i}{w}+b\right)\geqslant1*-\xi_i*
$$

Intuitively, $\xi_i$ is a measure of how much the constraint is violated:

- If *$0<\xi_i<1$*, then the $i$-th point is classified correctly, but is within the margin.
- If *$1<\xi_i<2$*, then the $i$-th point is misclassified and is within the margin.
- If *$\xi_i>2$*, then the $i$-th point is misclassified and is beyond the margin.

The objective will therefore be to minimize the sum of $\norm{\boldsymbol{w}}^2$ and the average of the stack variables. We understand then that the squared norm of the weight vector will assume the role of a stabilizer, therefore it will be multiplied by a parameter $\lambda$ that will regulate the fitting-stability trade-off:

$$
    \boxed{\min_{\boldsymbol{w},b,\xi}\left(\frac{1}{m}\sum_{i=1}^m\xi_i+\lambda\norm{\boldsymbol{w}}^2\right) \ \hspace{0.65cm} \text{with} \ \hspace{0.35cm} y_i\left(\sprod{x_i}{w}+b\right)\geqslant1-\xi_i,\xi_i\geqslant0 \ \hspace{0.35cm} \forall i}
$$

Let's pay attention to the fact that here it is not about the $\argmin$ function but about the $\min$ function, as the objective is to minimize the sum itself also with respect to *all* the slack variables $\xi_i$ which are not provided for in the final output: the latter, in fact, will simply be equal to $(\boldsymbol{w},b)$. *(This does not explain, however, why we include $b$ in the search for the minimum if no $b$ is present in the expression; although I do not know how to justify it, I report it anyway because maybe I'm missing something.)*

Let's see how the result changes as a function of $\lambda$: *(The professor tried to explain the two cases, getting confused three times in two minutes, and still ending up saying something different from what was written on the slide. I have tried to explain the two extremes in my own words as I understand the situations.)*

- $\boldsymbol{\lambda}$ **large**: the term $\lambda\norm{\boldsymbol{w}}^2$ will dominate the function, so the model will give great importance to keeping $\norm{\boldsymbol{w}}$ small, tending to choose a vector $\boldsymbol{w}$ with a very low norm. This leads to a wider margin between the classes at the cost of increasing the number of classification errors (expressed with higher values of $\xi_i$). Due to the underestimation of errors, the model could be subject to *underfitting*.
    
- $\boldsymbol{\lambda}$ **small**: the error penalty term $\frac{1}{m}\sum\limits_{i=1}^m\xi_i$ will dominate the function, so the model will give more weight to minimizing classification errors rather than maintaining a large margin. This means that the model will be willing to tolerate larger values of $\norm{\boldsymbol{w}}$ in order to minimize the $\xi_i$. Since the model could overfit the data to minimize the margin, there is a risk of *overfitting*.

### Reformulation with the hinge loss

Since we have rewritten the function with the stabilizer, we can try to rewrite the other piece in the form of a *loss function*. We would therefore want something like

$$
    \min_{\boldsymbol{w},b}\left(L_{\mathcal{S}}(\boldsymbol{w},b)+\lambda\norm{\boldsymbol{w}}^2\right)=\min_{\boldsymbol{w},b}\left(\frac{1}{m}\sum_{i=1}^m\ell\left((\boldsymbol{w},b),\left(\boldsymbol{x_i},y_i\right)\right)+\lambda\norm{\boldsymbol{w}}^2\right)
$$

Let's go back to the inequality @eq:support-vector-machine-soft-svm-variabili-slack and consider the extreme case of equality. Isolating $\xi_i$ we get:

$$
    \xi_i=1-y_i\left(\sprod{x_i}{w}+b\right)
$$

Since if the error is classified correctly we want the loss to be zero, it will result:

$$
    \ell^{hinge}\left((\boldsymbol{w},b),(\boldsymbol{x},y)\right)=\max\left(0,1-y(\sprod{x}{w}+b)\right)
$$

This loss function, as it is written, is called **hinge loss**\footnote{The term *hinge* means *zipper*, as its graph resembles the zipper of a door: the function is in fact linearly decreasing up to $1-y(\sprod{w}{x}+b)=1$ and then constant and flat at ordinate $0$.}

For correctly classified points, we have that $y(\sprod{x}{w}+b)>0$: in the case where $y(\sprod{x}{w}+b)>1$, they are both correctly classified and outside the margin, and in fact it results that $1-y(\sprod{x}{w}+b)<0$, from which it follows that $l^{hinge}=0$; in the case where $0<y(\sprod{x}{w}+b)<1$, the point is correctly classified but within the margin, and in fact the error becomes greater than zero, being equal to $0<\ell^{hinge}<1$. The last possibility is that $y(\sprod{x}{w}+b)<0$, a condition that occurs when the points are not correctly classified, and results in an even greater loss: $l^{hinge}>1$.

We therefore find the following formulation of the problem:

$$
    \boxed{\min_{\boldsymbol{w},b}\left(\frac{1}{m}\sum_{i=1}^m\max\left(0,1-y_i\left(\sprod{x_i}{w}+b\right)\right)+\lambda\norm{\boldsymbol{w}}^2\right)}
$$

We can formally prove that expressions @eq:support-vector-machine-soft-svm-prima-formulazione and @eq:support-vector-machine-soft-svm-problema-hinge-loss are equivalent.

::: {.theorem name="Equivalence of the expression with hinge loss"}
The expressions @eq:support-vector-machine-soft-svm-prima-formulazione and @eq:support-vector-machine-soft-svm-problema-hinge-loss are equivalent.

\exrule

Let's start from @eq:support-vector-machine-soft-svm-prima-formulazione and calculate the smallest values of $\xi_i$ for which the inequality is valid. First, let's express the inequality as a function of $\xi_i$:

$$
    \xi_i\geqslant1-y_i\left(\sprod{x_i}{w}+b\right)
$$

We distinguish two cases:


    - $y_i\left(\sprod{x_i}{w}+b\right)\geqslant1$
    
    This condition implies that $1-y_i\left(\sprod{x_i}{w}+b\right)\leqslant0$, so on the right-hand side of @eq:support-vector-machine-soft-svm-hinge-loss-teorema we have a negative value. Recalling that the other constraint is $\xi_i\geqslant0$, then the smallest value for which the inequality is valid is $\xi_i=0$.

    - $y_i\left(\sprod{x_i}{w}+b\right)<1$
    
    This condition implies that $1-y_i\left(\sprod{x_i}{w}+b\right)>0$, so on the right-hand side of @eq:support-vector-machine-soft-svm-hinge-loss-teorema we have a positive value. The smallest value for which @eq:support-vector-machine-soft-svm-hinge-loss-teorema is valid is therefore the one obtained from the case of equality, that is $\xi_i=1-y_i\left(\sprod{x_i}{w}+b\right)$.


We have identified the following function:

$$
    \xi_i=
    \begin{cases}
        1-y_i\left(\sprod{x_i}{w}+b\right) \ \  &y_i\left(\sprod{x_i}{w}+b\right)<1\\
        0 \ \   &y_i\left(\sprod{x_i}{w}+b\right)\geqslant1
    \end{cases}
$$

We can rewrite it by bringing all the conditions on the right to the first member:

$$
    \xi_i=
    \begin{cases}
        0 \ \   &1-y_i\left(\sprod{x_i}{w}+b\right)\leqslant0\\
        1-y_i\left(\sprod{x_i}{w}+b\right) \ \  &1-y_i\left(\sprod{x_i}{w}+b\right)>0
    \end{cases}
$$

From the latter it clearly emerges that

$$
    \xi_i=\max\left(0, 1-y_i\left(\sprod{x_i}{w}+b\right)\right)
$$

We can substitute this expression in @eq:support-vector-machine-soft-svm-prima-formulazione, omitting the conditions, as they are already included as we have seen. By doing so, we obtain exactly @eq:support-vector-machine-soft-svm-problema-hinge-loss, thus proving the thesis.
:::

Whether one formulation or the other is considered, it is in any case a minimization problem, which is usually approached iteratively through an algorithm that we will see later, the *Stochastic Gradient Descent*.
