14  Principal Component Analysis

14.1 Motivation

In many applications, we collect data in a high-dimensional space where \(d\) is very large (hundreds or thousands of dimensions). However, the intrinsic dimensionality of the data—the minimal number of dimensions needed to describe its meaningful variation—is often much smaller. Dimensionality reduction tackles this problem by projecting data from \(\mathbb{R}^d\) into a lower-dimensional space \(\mathbb{R}^n\) (where \(n \ll d\)) while preserving as much structure and variance as possible.

The primary motivations are:

  • Computational efficiency: Fewer dimensions mean faster training and inference.
  • Generalization: Reducing dimensions combats the curse of dimensionality and can improve learning performance.
  • Interpretability: Visualizing data in 2D or 3D, or extracting meaningful structure.
  • Compression: Storing and transmitting data more efficiently.

14.2 Problem Setup

We have \(m\) data points \(x_1, x_2, \ldots, x_m \in \mathbb{R}^d\). We seek two linear transformations:

  • A compression (encoder) \(W \in \mathbb{R}^{n \times d}\) that maps \(x_i \mapsto y_i = Wx_i \in \mathbb{R}^n\),
  • A reconstruction (decoder) \(U \in \mathbb{R}^{d \times n}\) that maps \(y_i \mapsto \tilde{x}_i = Uy_i \approx x_i\).

The goal is to minimize the total reconstruction error:

\[\min_{W, U} \sum_{i=1}^{m} \|x_i - UWx_i\|_2^2.\]

14.3 Optimal Form of Reconstruction

Before solving this optimization, we characterize what the optimal solution must look like.

If \((U, W)\) is a solution to the above minimization problem, then:

  1. The columns of \(U\) are orthonormal: \(U^\top U = I_n\) (the \(n \times n\) identity).
  2. \(W = U^\top\).

Intuition: The optimal reconstruction projects onto an orthonormal basis, then uses its transpose to recover. Any non-orthonormal choice either wastes degrees of freedom or fails to optimally reconstruct.

Using this lemma, we can rewrite the optimization as:

\[\min_{U \in \mathbb{R}^{d \times n} : U^\top U = I_n} \sum_{i=1}^{m} \|x_i - UU^\top x_i\|_2^2.\]

14.4 Simplification via the Covariance Matrix

For any orthonormal \(U\) and any \(x\), we have:

\[\|x - UU^\top x\|_2^2 = \|x\|_2^2 - \text{trace}(U^\top xx^\top U).\]

Therefore, minimizing reconstruction error is equivalent to:

\[\max_{U \in \mathbb{R}^{d \times n} : U^\top U = I_n} \text{trace}\left(U^\top \left(\sum_{i=1}^{m} x_i x_i^\top\right) U\right).\]

Define the covariance matrix (or outer-product matrix):

\[A = \sum_{i=1}^{m} x_i x_i^\top \in \mathbb{R}^{d \times d}.\]

We want to maximize \(\text{trace}(U^\top AU)\) subject to \(U^\top U = I_n\).

14.5 Main Theorem: PCA Solution

Let \(x_1, \ldots, x_m \in \mathbb{R}^d\) and \(A = \sum_{i=1}^{m} x_i x_i^\top\). Let \(u_1, \ldots, u_n\) be the eigenvectors of \(A\) corresponding to its \(n\) largest eigenvalues (in descending order). Then, the optimal compression and reconstruction matrices are:

  • Compression: \(W = U^\top\) where \(U = [u_1 \mid u_2 \mid \cdots \mid u_n]\) (columns are the top-\(n\) eigenvectors).
  • Reconstruction: The projection matrix is \(U^\top U\).

The optimal value of the objective is \(\sum_{j=1}^{n} \lambda_j\), where \(\lambda_1 \geq \lambda_2 \geq \cdots \geq \lambda_d\) are the eigenvalues of \(A\) sorted in descending order.

Intuition: The directions of maximum variance are the principal components. By projecting onto these directions, we capture the most information about the data with the fewest dimensions. The variance explained by the \(j\)-th principal component is precisely \(\lambda_j\).

14.6 Reconstruction Error and Variance Explained

The total squared reconstruction error is:

\[\text{Error} = \sum_{i=1}^{m} \|x_i - UU^\top x_i\|_2^2 = \sum_{j=n+1}^{d} \lambda_j.\]

This equals the sum of the discarded eigenvalues—the variance in the directions we ignore.

The variance explained by the first \(n\) principal components is:

\[\text{Variance Explained} = \frac{\sum_{j=1}^{n} \lambda_j}{\sum_{j=1}^{d} \lambda_j} = \frac{\sum_{j=1}^{n} \lambda_j}{\text{trace}(A)}.\]

This quantity guides practitioners in choosing \(n\): typically, we select \(n\) to capture 90-95% of the total variance.

14.7 Computational Efficiency: The SVD Trick

When the number of data points \(m\) is much smaller than the dimensionality \(d\) (i.e., \(m \ll d\)), computing \(A = X^\top X \in \mathbb{R}^{d \times d}\) is expensive. We can instead work with:

\[B = XX^\top \in \mathbb{R}^{m \times m}.\]

Remark. If \(v\) is an eigenvector of \(B\) with eigenvalue \(\lambda\), then \(w = X^\top v / \|X^\top v\|\) is an eigenvector of \(A\) with the same eigenvalue \(\lambda\). Thus, we can compute PCA by:

  1. Computing eigendecomposition of \(B\) (cost \(O(m^3)\)).
  2. Scaling the eigenvectors: \(u_j = X^\top v_j / \|X^\top v_j\|\).

This reduces complexity from \(O(d^3)\) to \(O(m^3)\) when \(m \ll d\).

14.8 Kernel PCA

A natural extension uses kernels to perform nonlinear dimensionality reduction implicitly. Instead of working with \(x_i^\top x_j\), we replace it with \(K(x_i, x_j)\) for some kernel function (e.g., RBF, polynomial). The algorithm remains the same but now performs PCA in an implicit high-dimensional (or even infinite-dimensional) feature space. This allows capturing nonlinear structure without explicitly computing the feature transformation.

14.9 Applications

14.9.1 Eigenfaces for Face Recognition

PCA was famously applied to face images:

Given a set of 50×50 pixel face images (dimensionality \(d = 2500\)), we compute PCA and retain, say, \(n = 100\) principal components. The eigenvectors look like “ghost” faces and are called eigenfaces. Each face is then represented as a 100-dimensional vector of coefficients, enabling fast similarity search and dimensionality reduction for face recognition. Despite the simplicity, this method was state-of-the-art in the 1990s.

14.9.2 2D Visualization

When \(d\) is large, we project data onto the first two principal components \(u_1\) and \(u_2\) to visualize high-dimensional structure in the plane. This often reveals clusters, outliers, and relationships not obvious in the original space.

14.9.3 Feature Extraction

Rather than using pixels directly, practitioners precompute PCA features and feed the lower-dimensional representation to a classifier (SVM, neural network, etc.). This speeds up training and sometimes improves generalization by noise reduction.

14.10 Practical Considerations

  1. Centering: It is standard to center the data by subtracting the mean \(\mu = \frac{1}{m} \sum_{i=1}^m x_i\) before applying PCA. This makes \(A\) the sample covariance matrix, aligning PCA with variance maximization.

  2. Standardization: If features have different scales, standardize to unit variance first.

  3. Choosing \(n\): Use the cumulative variance explained plot. Plot \(\frac{\sum_{j=1}^n \lambda_j}{\sum_{j=1}^d \lambda_j}\) versus \(n\); choose \(n\) where the curve flattens (e.g., 90% threshold).

  4. Out-of-sample projection: To project new data points \(x_{\text{new}}\), compute \(y_{\text{new}} = U^\top (x_{\text{new}} - \mu)\).

14.11 Example: 2×2 Case

Consider four points: \((1, 0)\), \((0, 1)\), \((1, 1)\), \((0, 0)\). Centering gives mean \((1/2, 1/2)\), so the centered points are: \[(1/2, -1/2), (-1/2, 1/2), (1/2, 1/2), (-1/2, -1/2).\]

The covariance matrix is: \[A = \frac{1}{4}\begin{pmatrix} 1/4 + 1/4 + 1/4 + 1/4 & \text{(cross terms)} \\ \vdots & \vdots \end{pmatrix} = \begin{pmatrix} 1/2 & 0 \\ 0 & 1/2 \end{pmatrix}.\]

Eigenvalues are \(\lambda_1 = \lambda_2 = 1/2\). The variance is equally distributed, so both components are equally important. If we project to 1D, we must choose a direction (e.g., along the first eigenvector); reconstruction error is \(1/2\).

## Motivation

$$\min_{W, U} \sum_{i=1}^{m} \|x_i - UWx_i\|_2^2.$$

$$B = XX^\top \in \mathbb{R}^{m \times m}.$$
