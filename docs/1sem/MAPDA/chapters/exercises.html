Using bit-masking operations, invert the seventh bit (starting from the most significant one) of a 32-bit register \(R_0\).

Solution. To invert a bit using bit-masking it is necessary to perform a XOR with a mask composed only of 0s except for a 1 in the position of the bit. Since this is in the seventh position from the left, it will be located in the second hexadecimal digit of the mask:

\[ 0010=0\text{x}2 \]

Since there are 32 bits, the hexadecimal digits will be a total of \(8\), all equal to \(0\). The operation, therefore, will be the following:

\[ R_0 \ \textit{XOR} \ 0\text{x}02000000 \]

What does the following program for SAP-3 do? First, the extended register HL is loaded with the data \(0\)x\(2501\): \[HL=0\text{x}2501\] The content of the memory indicated by HL is copied into C and A: \[A=C=R_{0\text{x}2501}\] The extended register HL is incremented, thus pointing to the next memory address; the content of the latter is subtracted from the content in the accumulator, which is the same content as the previous address: \[A=R_{0\text{x}2501}-R_{0\text{x}2502}\] The sign flag is therefore activated as follows: \[S= \begin{cases} 0 \ \ \text{if} \ R_{0\text{x}2501}\geqslant R_{0\text{x}2502}\\ 1 \ \ \text{if} \ R_{0\text{x}2501}<R_{0\text{x}2502} \end{cases}\] The content of \(R_{0\text{x}2501}\) is copied again into the accumulator: \[A=R_{0\text{x}2501}\]

If the sign flag is null, the content of the accumulator, and therefore of \(R_{0\text{x}2501}\), is saved in the address \(R_{0\text{x}2503}\); if instead the sign flag is set, the content of the address to which the data pointer points, which is always \(R_{0\text{x}2502}\), is copied into A, and it is this that is saved in \(R_{0\text{x}2503}\).

In summary, the program saves in the address \(R_{0\text{x}2503}\) the largest content between that contained in the two previous addresses.

::: {.exercise name="Worked exercise 5 — 23/03/23"}
*What does the following program for SAP-3 do?*
First, the extended register *HL* is loaded with the data $0$x$2501$:
$$HL=0\text{x}2501$$
The content of the memory indicated by *HL* is copied into *C* and *A*:
$$A=C=R_{0\text{x}2501}$$
The extended register *HL* is incremented, thus pointing to the next memory address; the content of the latter is subtracted from the content in the accumulator, which is the same content as the previous address:
$$A=R_{0\text{x}2501}-R_{0\text{x}2502}$$
The sign flag is therefore activated as follows:
$$S=
        \begin{cases}
            0 \ \ \text{if} \ R_{0\text{x}2501}\geqslant R_{0\text{x}2502}\\
            1 \ \ \text{if} \ R_{0\text{x}2501}<R_{0\text{x}2502}
        \end{cases}$$
The content of $R_{0\text{x}2501}$ is copied again into the accumulator:
$$A=R_{0\text{x}2501}$$

If the sign flag is null, the content of the accumulator, and therefore of $R_{0\text{x}2501}$, is saved in the address $R_{0\text{x}2503}$; if instead the sign flag is set, the content of the address to which the data pointer points, which is always $R_{0\text{x}2502}$, is copied into *A*, and it is this that is saved in $R_{0\text{x}2503}$.

In summary, the program saves in the address $R_{0\text{x}2503}$ the largest content between that contained in the two previous addresses.
